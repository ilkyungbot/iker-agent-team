# UX 원칙 — UX 원칙과 휴리스틱
> 핵심 메시지: "좋은 UX는 사용자가 인식하지 못한다. 나쁜 UX는 사용자가 욕한다"

## 핵심 원칙
- 사용자 멘탈 모델에 맞는 인터페이스를 설계한다
- 오류를 예방하는 것이 오류 메시지보다 낫다
- 단순함이 가장 어렵다: 복잡성을 줄이는 것이 핵심
- 사용자 테스트로 가정을 검증한다

## 판단 기준
- 이 UI를 처음 보는 사용자가 다음 행동을 알 수 있는가?
- 오류 발생 시 사용자가 스스로 복구할 수 있는가?
- 가장 중요한 액션이 가장 눈에 띄는가?
- 사용자가 학습 없이도 사용할 수 있는가?

## 실전 예시 (Nielsen의 10대 휴리스틱)

### 10대 사용성 휴리스틱
1. **시스템 상태 가시성**: 사용자에게 무슨 일이 일어나는지 알려라
   - 예: 파일 업로드 진행률 표시
2. **실세계와 시스템 일치**: 사용자 언어와 개념을 사용하라
   - 예: 기술 용어 대신 일상 언어
3. **사용자 제어와 자유**: 실수 복구 수단을 제공하라
   - 예: 삭제 취소 기능
4. **일관성과 표준**: 같은 것은 같게 표현하라
   - 예: 저장 버튼은 항상 같은 위치
5. **오류 예방**: 문제 발생 전에 방지하라
   - 예: 폼 제출 전 유효성 검사
6. **인식 vs 기억**: 기억보다 인식하게 만들라
   - 예: 드롭다운으로 옵션 표시
7. **유연성과 효율**: 초보자와 전문가 모두 지원하라
   - 예: 키보드 단축키 제공
8. **미적이고 단순한 디자인**: 불필요한 요소를 제거하라
   - 예: 불필요한 텍스트와 아이콘 제거
9. **오류 인식, 진단, 복구 지원**: 명확한 오류 메시지를 제공하라
   - 예: "에러 발생" 대신 "이메일 형식이 올바르지 않습니다"
10. **도움말과 문서**: 필요시 도움을 쉽게 찾을 수 있게 하라

### JTBD 기반 UX 설계 원칙
- 목적 달성에 불필요한 단계 제거
- 사용자의 현재 상황 맥락을 인터페이스에 반영
- 가장 빈번한 태스크를 가장 쉽게 접근 가능하게

### 인지 부하 감소 전략
- Progressive Disclosure: 필요할 때만 정보 표시
- Chunking: 정보를 의미 있는 그룹으로 묶기
- Default Values: 가장 일반적인 선택을 기본값으로
- Recognition over Recall: 기억보다 인식이 쉬운 UI

### 접근성 기본 체크리스트
- [ ] 색상만으로 정보를 전달하지 않음
- [ ] 충분한 색상 대비 (WCAG AA: 4.5:1)
- [ ] 키보드로만 모든 기능 사용 가능
- [ ] 스크린 리더를 위한 alt text 제공
- [ ] 클릭 영역 최소 44x44px

## 안티패턴
- Dark Pattern: 사용자를 속이거나 원치 않는 행동 유도
- 클릭 유도를 위한 혼란스러운 디자인
- "회원가입 없이 계속하기" 버튼을 찾기 어렵게 배치
- 오류 메시지를 어디서 발생했는지 알 수 없게 표시
- 모바일을 나중에 생각하는 Desktop-First 설계

## 실전 팁
- 5-second test: 5초 안에 이 페이지의 목적을 이해할 수 있는가?
- 경쟁사 UX 분석을 월 1회 정기적으로 수행
- UX 리뷰를 스프린트 사이클에 포함
- 사용자 테스트는 5명이면 주요 문제의 85% 발견
- "할머니 테스트": 비기술적 사용자도 사용 가능한가?

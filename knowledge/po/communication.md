# 커뮤니케이션 — 이해관계자 커뮤니케이션
> 핵심 메시지: "PO의 절반은 제품을 만들고, 나머지 절반은 제품을 설명하는 데 쓴다"

## 핵심 원칙
- 수신자 중심 커뮤니케이션: 상대방의 관심사와 언어를 사용
- 구조화된 커뮤니케이션이 설득력을 높인다
- 나쁜 소식은 빨리, 좋은 소식은 증거와 함께
- "왜"를 먼저, "무엇"을 나중에: Pyramid Principle

## 판단 기준
- 이 메시지가 수신자의 의사결정에 도움이 되는가?
- 핵심 메시지가 30초 내에 전달되는가?
- 데이터와 근거가 주장을 뒷받침하는가?
- 다음 액션이 명확한가?

## 실전 예시 (프레임워크)

### Pyramid Principle (바바라 민토)
1. 결론 먼저 (So what?)
2. 이유 3가지 (Why?)
3. 각 이유의 근거 (Evidence)

예시:
- 결론: "이번 분기 결제 기능 개선을 최우선으로 해야 합니다"
- 이유1: 결제 전환율 15% 낮음 (업계 평균 대비)
- 이유2: 사용자 인터뷰 8건에서 결제 불편 언급
- 이유3: 개선 시 MRR 20% 증가 예상

### SCQA 스토리텔링 구조
- **S**ituation: 상황 설명 (공통 기반)
- **C**omplication: 문제/긴장감 (왜 변화가 필요한가)
- **Q**uestion: 핵심 질문 (무엇을 해야 하는가)
- **A**nswer: 해결책 (우리의 제안)

### 이해관계자별 커뮤니케이션 스타일
| 이해관계자 | 관심사 | 소통 방식 |
|---------|------|---------|
| CEO | 비즈니스 임팩트, 전략 | 1-pager, 핵심 지표 |
| CTO | 기술 실현 가능성, 부채 | 기술 상세, 트레이드오프 |
| 영업팀 | 고객 약속 가능 기능 | 로드맵, 타임라인 |
| 디자이너 | 사용자 경험, 일관성 | UX 플로우, 사용자 조사 |
| 고객 | 문제 해결, 사용 편의 | 데모, 출시 노트 |

### 어려운 대화 처리법
상황: 이해관계자 요청을 거절해야 할 때

3단계 접근:
1. 공감 표현: "이 기능이 왜 중요한지 이해합니다"
2. 현재 상황 설명: "현재 우선순위는 X입니다. 이유는..."
3. 대안 제시: "이번 분기는 어렵지만, Q3에 검토할 수 있습니다"

### 주요 커뮤니케이션 채널별 형식
- Slack/채팅: 비격식, 빠른 업데이트, 짧게
- 이메일: 공식 커뮤니케이션, 결정 사항 기록
- 주간 업데이트: What, So What, Now What
- All-Hands: 비전 연결, 성과 공유, Q&A
- 스프린트 리뷰: 데모 중심, 피드백 수집

### 주간 업데이트 템플릿
```
📊 이번 주 주요 지표
- [지표 1]: X → Y (±Z%)
- [지표 2]: X → Y

✅ 완료된 것
- [항목 1]
- [항목 2]

🔄 진행 중
- [항목 1] (예상 완료: X주)

🚧 블로커
- [이슈]: 해결 방안 [X]

📅 다음 주 계획
- [항목 1]
```

## 안티패턴
- 정보 과부하: 모든 것을 공유하려는 욕심
- 수동적 커뮤니케이션: 묻기 전까지 업데이트 안 함
- 나쁜 소식 숨기기: 나중에 더 큰 문제로 돌아옴
- 이해관계자마다 다른 메시지: 일관성 없는 커뮤니케이션
- 방어적 태도: 피드백을 공격으로 받아들임

## 실전 팁
- "요약하면 ~입니다"로 대화를 시작하는 습관
- 중요한 결정은 문서로 남기고 링크 공유
- 회의 전 아젠다, 회의 후 액션 아이템 발송
- 불확실한 것은 "모릅니다"가 "예상됩니다"보다 신뢰를 준다
- 분기별 이해관계자 1:1 미팅으로 관계 유지

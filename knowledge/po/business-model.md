# 비즈니스 모델 — 비즈니스 모델 분석과 설계
> 핵심 메시지: "좋은 제품도 나쁜 비즈니스 모델 위에서는 살아남지 못한다"

## 핵심 원칙
- 비즈니스 모델은 가치 창출 + 전달 + 포획의 시스템
- 수익화 방식이 사용자 행동에 영향을 준다 (정렬 중요)
- 단위 경제학(Unit Economics)이 맞아야 스케일 가능
- 비즈니스 모델 혁신이 기능 혁신보다 강력할 때가 많다

## 판단 기준
- LTV > CAC × 3인가? (건강한 단위 경제학)
- 수익화 방식이 고객 가치 제공과 정렬되어 있는가?
- 규모가 커질수록 수익성이 개선되는가? (규모의 경제)
- 경쟁사가 쉽게 복제할 수 없는 수익 구조인가?

## 실전 예시 (프레임워크)

### Business Model Canvas (9 요소)
| 요소 | 핵심 질문 |
|-----|---------|
| 고객 세그먼트 | 누구를 위한 가치를 창출하는가? |
| 가치 제안 | 어떤 가치를 제공하는가? |
| 채널 | 어떻게 고객에게 도달하는가? |
| 고객 관계 | 어떤 관계를 유지하는가? |
| 수익 흐름 | 어떻게 수익을 창출하는가? |
| 핵심 자원 | 어떤 자원이 필요한가? |
| 핵심 활동 | 어떤 활동이 핵심인가? |
| 핵심 파트너십 | 어떤 파트너가 필요한가? |
| 비용 구조 | 주요 비용은 무엇인가? |

### SaaS 비즈니스 모델 핵심 지표
- MRR (Monthly Recurring Revenue): 월 반복 매출
- ARR (Annual Recurring Revenue): 연 반복 매출
- Churn Rate: 월간 이탈률 (목표 < 2%)
- NRR (Net Revenue Retention): 기존 고객 매출 유지율 (목표 > 100%)
- CAC (Customer Acquisition Cost): 고객 획득 비용
- LTV (Lifetime Value): 고객 생애 가치
- LTV/CAC 비율: 건강 기준 > 3
- Payback Period: CAC 회수 기간 (목표 < 12개월)

### 수익화 모델 유형
| 모델 | 특징 | 적합한 상황 |
|-----|------|-----------|
| Subscription | 예측 가능한 수익 | 반복 사용 서비스 |
| Freemium | 대규모 사용자 기반 | 네트워크 효과 |
| Usage-based | 가치 기반 과금 | 인프라, API |
| Transaction Fee | 거래 시 수수료 | 마켓플레이스 |
| Enterprise License | 대형 고객 계약 | B2B 복잡 제품 |
| Ad-based | 무료 사용자 규모 | 콘텐츠, 소셜 |

### 단위 경제학 계산
```
CAC = 마케팅+영업 비용 / 신규 고객 수

LTV = ARPU / Churn Rate
또는
LTV = ARPU × 평균 고객 수명

LTV/CAC = [목표 3 이상]
Payback = CAC / (ARPU × Gross Margin %)
```

### 가격 전략 프레임워크
- Cost-plus: 원가 + 마진 (단순하지만 가치 무시)
- Competitor-based: 경쟁사 기준 (레드오션)
- Value-based: 고객이 인지하는 가치 기준 (권장)
- Penetration: 낮은 가격으로 시장 진입 후 인상
- Skimming: 높은 가격으로 시작 후 인하

## 안티패턴
- LTV < CAC: 고객 유치할수록 손해
- 가격을 전략 없이 책정 (경쟁사 보고 베끼기)
- 수익화를 나중으로 미루기: "먼저 성장하고 나중에 수익화"
- 모든 기능을 무료로 제공: 수익화 전환 어려움
- 단위 경제학 추적 안 함: 성장할수록 더 큰 손실

## 실전 팁
- 월 1회 단위 경제학 지표 리뷰
- 가격 인상 실험: 신규 고객부터 적용하여 영향 측정
- 고객 세그먼트별 가격 민감도 분석
- Pricing Page A/B 테스트: 패키징 방법 최적화
- "이 비즈니스가 $100M ARR이 되려면 무엇이 달라야 하는가?" 역산

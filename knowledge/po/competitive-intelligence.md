# 경쟁사 분석 — 경쟁 인텔리전스 수집과 활용
> 핵심 메시지: "경쟁사를 아는 것은 자신을 아는 것만큼 중요하다. 단, 따라가기 위함이 아닌 차별화를 위해"

## 핵심 원칙
- 경쟁사 분석의 목적은 모방이 아닌 차별화 기회 발굴
- 직접 경쟁사보다 대체재(alternatives)가 더 중요할 때 있음
- 고객이 경쟁사 대신 우리를 선택하는 이유를 명확히
- 정기적 업데이트 없는 경쟁사 분석은 의미 없음

## 판단 기준
- 우리 고객이 실제로 사용하는 대안이 무엇인가?
- 경쟁사 대비 우리의 차별화 포인트가 명확한가?
- 경쟁사의 어떤 움직임이 우리 전략에 영향을 주는가?
- 경쟁 분석이 실제 제품 결정에 활용되는가?

## 실전 예시 (프레임워크)

### 경쟁 환경 분류
- 직접 경쟁사: 동일 문제, 유사 솔루션
- 간접 경쟁사: 동일 문제, 다른 솔루션
- 잠재 경쟁사: 진입 가능성 있는 플레이어
- 대체재: 고객이 현재 사용하는 모든 방법 (Excel, 수작업 포함)

### 경쟁사 프로파일 템플릿
```
회사명:
설립/규모: 직원 수, 자금 조달
타겟 고객:
핵심 가치 제안:
주요 기능:
가격 모델:
강점:
약점:
최근 동향 (출시, 자금조달, 파트너십):
우리 대비 포지션:
마지막 업데이트 날짜:
```

### 경쟁사 정보 수집 소스
- 제품 직접 사용 (Trial/Free 버전)
- 공개 문서: 웹사이트, 블로그, 도움말 센터
- 리뷰 사이트: G2, Capterra, 앱스토어 리뷰
- 채용 공고: 기술 스택, 성장 방향 파악
- 뉴스/PR: TechCrunch, LinkedIn 발표
- 고객 인터뷰: "다른 서비스도 사용하시나요?"
- 직원 LinkedIn: 조직 구조, 성장 패턴

### 포지셔닝 맵 작성
1. 업계 핵심 가치 축 2개 선정
2. 경쟁사 위치 매핑
3. 고객이 인식하는 우리 위치 확인 (인터뷰)
4. 목표 포지션 설정 및 갭 분석

### Win/Loss 분석
고객 획득 시: "왜 우리를 선택했나요?"
고객 이탈 시: "왜 다른 서비스로 이동했나요?"
공통 패턴 파악 → 차별화 강화 또는 약점 보완 결정

## 안티패턴
- 경쟁사 기능 따라가기: 차별화 포기
- 경쟁사 분석을 한 번만 하고 업데이트 안 함
- 경쟁사가 없다는 착각: 항상 대체재는 존재
- 경쟁사 움직임에 과잉 반응: 전략적 일관성 유지 필요
- 기능 비교만: 가격, UX, 브랜드, 지원 등 전체 경험 비교 필요

## 실전 팁
- 월 1회 주요 경쟁사 프로파일 업데이트
- 팀 전체가 경쟁사 제품을 직접 사용해 보는 세션 (분기별)
- 경쟁사 뉴스레터 구독, LinkedIn 팔로우
- 경쟁사 Job Description으로 기술 방향 예측
- "우리만이 할 수 있는 것" 리스트 주기적 검토

# 대시보드 설계 — 의사결정을 돕는 대시보드 만들기

> 대시보드는 데이터를 보여주는 것이 아니라 질문에 답하는 것이다.

## 핵심 원칙
- 대상 독자를 먼저 정의하라 (경영진 vs 실무자 vs 데이터팀)
- 하나의 대시보드는 하나의 질문에 답해야 한다
- 5초 규칙: 5초 안에 핵심 메시지를 파악할 수 있어야 한다
- 인터랙션보다 레이아웃이 먼저다
- 데이터 신선도(Last Updated)를 항상 표시하라

## 판단 기준
| 독자 유형 | 필요한 것 | 적합한 형태 |
|-----------|-----------|-------------|
| C-Level | 지금 상황이 좋은가? | KPI 요약 + 트렌드 |
| 팀장 | 왜 이렇게 됐나? | 드릴다운 가능한 차트 |
| 실무자 | 오늘 무엇을 해야 하나? | 상세 테이블 + 필터 |
| 데이터팀 | 데이터가 정상인가? | 품질 지표 + 이상 탐지 |

## 실전 예시

### 대시보드 구조 설계 원칙
```
1. 상단: 핵심 KPI 카드 (3-5개) + 목표 대비 달성률
   - 오늘 매출 / MoM / 목표 대비
   - 활성 사용자 / WoW
   - 전환율 / 트렌드 방향

2. 중단: 트렌드 차트 (시계열)
   - 주요 지표 30일 추이
   - 이상 탐지 기준선 표시

3. 하단: 세그먼트 분해
   - 채널별 / 지역별 / 상품별
   - 드릴다운 가능하게

4. 사이드: 필터 패널
   - 날짜 범위 / 세그먼트
```

### Looker Studio / Metabase 설계 체크리스트
```
레이아웃:
□ 가장 중요한 숫자가 좌상단 또는 최상단
□ 관련 차트는 그룹핑
□ 여백 충분 (숨막히지 않게)
□ 색상 3가지 이하 (강조색 1개)

데이터:
□ 지표 정의 툴팁 또는 설명 포함
□ 이전 기간 비교 수치 포함
□ 데이터 기준일/갱신 시간 표시
□ 이상치/목표값 기준선 표시

인터랙션:
□ 날짜 필터 (기본값: 최근 30일)
□ 핵심 세그먼트 필터
□ 상세 데이터 다운로드 옵션
```

### 색상 사용 원칙
```
시맨틱 색상:
- 초록: 목표 달성 / 긍정 트렌드
- 빨강: 목표 미달 / 부정 트렌드
- 파랑: 중립 / 기본 데이터
- 주황: 주의 / 경계값

비교 차트:
- 기간 비교: 진한색(현재) + 연한색(이전)
- 세그먼트: 색조 구분 (같은 색상군 내)
- 강조: 하나만 강조색, 나머지는 회색
```

## 안티패턴
- 차트 20개짜리 슈퍼 대시보드 — 아무것도 안 보임
- 목표치 없이 절댓값만 표시 — 좋은지 나쁜지 모름
- 3D 파이차트 — 면적 비교 불가, 시각적 왜곡
- 자동 갱신 없는 대시보드 — "이게 최신 데이터냐"는 질문 반복
- 필터 기본값 미설정 — 열 때마다 전체 기간 조회 → 느림

## 실전 팁
- 대시보드 리뷰 미팅: 숫자 읽기가 아니라 인사이트 토론으로
- 알림(Alert) 설정: 이상 탐지 자동화 → 대시보드를 매일 볼 필요 없게
- 대시보드 deprecation: 6개월 미조회 시 아카이브
- 모바일 최적화: C-Level은 스마트폰으로 볼 가능성 높음
- "So What?" 테스트: 모든 차트에 행동 가능한 결론이 있는가?

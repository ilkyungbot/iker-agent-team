# 데이터 스토리텔링 — 데이터를 설득력 있는 이야기로

> 최고의 분석가는 데이터를 잘 다루는 사람이 아니라 이야기를 잘 하는 사람이다.

## 핵심 원칙
- 데이터는 증거이지 이야기가 아니다. 이야기는 사람이 만든다
- 감정을 자극하라 — 숫자는 기억에 남지 않지만 이야기는 남는다
- 갈등 구조: 현재 상황(문제) → 원인 → 해결책
- 데이터는 3개 이하로 집중 — 많으면 아무것도 기억 못 함
- 청중의 관심사에서 시작하라, 분석가의 발견에서 시작하지 말라

## 판단 기준
| 스토리 유형 | 구조 | 적합한 상황 |
|-------------|------|-------------|
| 탐정 이야기 | 이상 현상 → 단서 탐색 → 원인 발견 | 문제 원인 분석 |
| 변화 이야기 | Before → After → 무엇이 달라졌나 | A/B 테스트 결과 |
| 기회 이야기 | 현재 상황 → 갭 발견 → 성장 가능성 | 신규 기회 제안 |
| 경고 이야기 | 트렌드 → 미래 예측 → 지금 해야 할 것 | 위험 대응 촉구 |

## 실전 예시

### SCR 구조 (Situation-Complication-Resolution)
```
Situation (현황):
"3월 매출은 전월 대비 5% 성장하며 목표를 달성했습니다."

Complication (문제):
"하지만 신규 고객 매출은 15% 감소했고,
기존 고객의 대형 계약 덕분에 전체 수치가 유지된 것입니다."

Resolution (해결책):
"신규 고객 전환율 개선을 위해 온보딩 플로우를 개편하고,
다음 분기에 A/B 테스트를 진행할 것을 제안합니다."
```

### Pyramid 구조 (결론 먼저)
```
Top (결론):
"지금 당장 모바일 앱 온보딩을 개편해야 합니다."

Supporting argument 1:
"모바일 Day 1 리텐션이 데스크탑 대비 40% 낮습니다."
  - 데이터: 모바일 38% vs 데스크탑 52%
  - 기간: 최근 3개월

Supporting argument 2:
"모바일 이탈 사용자의 70%가 온보딩 3단계에서 이탈합니다."
  - 데이터: 퍼널 분석 결과
  - 이탈 지점: 권한 요청 화면

Supporting argument 3:
"경쟁사 A의 간소화된 온보딩 도입 후 리텐션 20% 개선 사례"
  - 출처: 공개 케이스 스터디
```

### 프레젠테이션 슬라이드 구조
```python
# 각 슬라이드 구조 (Assertion + Evidence)
slides = [
    {
        "title": "모바일 사용자가 빠르게 이탈하고 있습니다",  # 주장
        "chart": "모바일 vs 데스크탑 리텐션 추이 (선 그래프)",  # 증거
        "annotation": "4월부터 격차 확대 시작"  # 맥락
    },
    {
        "title": "문제는 온보딩 3단계 권한 요청 화면입니다",
        "chart": "온보딩 퍼널 (가로 막대 비교)",
        "annotation": "이탈률 68% → 3단계"
    },
    {
        "title": "권한 요청 타이밍 변경으로 즉시 개선 가능합니다",
        "chart": "유사 사례 Before/After",
        "annotation": "구현 기간: 1주일"
    }
]
```

## 안티패턴
- 분석 과정을 모두 보여주는 리포트 — 결론만 필요함
- "흥미롭게도..." 같은 분석가 시각으로 시작
- 모든 데이터를 공평하게 제시 — 메시지가 없어짐
- 한계점/불확실성을 숨기기 — 신뢰도 하락 위험
- 숫자만 나열하고 "알아서 판단하세요" — 의사결정 회피

## 실전 팁
- "So What?" 연습: 모든 데이터 포인트에 "그래서 무엇이 달라져야 하나?" 질문
- 1분 버전 먼저 만들기: 엘리베이터 피치로 요약 후 확장
- 청중 분석: 그들이 지금 가장 걱정하는 것은 무엇인가?
- 적의 입장 대비: "반론은 무엇인가? 데이터 한계는?"
- 이야기 리허설: 동료에게 먼저 설명하고 "이해됐나?" 확인

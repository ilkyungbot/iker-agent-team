# 사용성 테스트 — 실제 사용자로 검증하는 디자인

> 디자이너는 자신의 디자인을 테스트할 수 없다. 이미 정답을 알고 있기 때문이다.

## 핵심 원칙

- **과제 기반**: 자유 탐색보다 현실적인 과제(Task)를 부여한다
- **관찰자 역할**: 테스트 진행자는 도와주지 않는다. 관찰만 한다
- **5명의 법칙**: 정성 테스트 5명으로 주요 사용성 문제의 약 85%를 발견
- **프로토타입으로 충분**: 완성된 제품이 아니어도 사용성 테스트 가능
- **빠른 반복**: 테스트 → 수정 → 재테스트 사이클을 짧게 유지

## 판단 기준

**테스트 대상 선정 기준**
- 신규 사용자: 온보딩 흐름, 핵심 가치 전달
- 기존 사용자: 새로운 기능, 리디자인된 영역
- 특수 사용자: 장애인, 고령층, 저디지털 리터러시

**과제(Task) 좋은 예시**
- O "쇼핑몰에서 파란색 운동화를 구매해 보세요"
- X "필터 기능을 사용해 보세요" (기능 명칭 직접 언급 금지)
- O "저번 달 결제 내역을 확인하고 영수증을 저장해 보세요"

## 실전 예시

**테스트 진행 스크립트 구성**
1. 시작 멘트 (5분): 목적 설명, 편안한 분위기 조성, 녹화 동의
2. 워밍업 (5분): 배경 질문, 기기 사용 습관
3. 과제 수행 (20~30분): 3~5개 핵심 시나리오
4. 사후 인터뷰 (10분): 전반적 경험, 개선 제안
5. 마무리 (5분): 보상 제공, 감사 인사

**관찰 기록 항목**
- 과제 성공/실패 여부
- 소요 시간
- 오류 발생 횟수 및 위치
- 사용자 발화 (혼잣말 포함)
- 표정·행동 변화 포인트

## 안티패턴

- 막히면 힌트 제공 → 실제 사용성 데이터 오염
- 과제 성공/실패 이분법으로만 평가 → 과정 정보 손실
- 테스트 결과를 "나 잘했지?" 확인용으로 사용
- 디자이너가 직접 운영하면서 방어적 반응
- 테스트 후 개선 없이 다음 테스트로 넘어감

## 실전 팁

- Maze, UserTesting, Lookback 등 원격 테스트 툴 활용으로 빠른 리크루팅
- 테스트 진행 중 노트는 다른 팀원이 담당. 진행자는 참여자에 집중
- Rainbow Spreadsheet로 여러 참여자 패턴을 한눈에 파악
- 테스트 클립 영상을 PM/개발자와 공유 → 공감 극대화
- 개선 우선순위: 빈도(몇 명이 겪었나) × 심각도(얼마나 방해됐나)로 매트릭스 작성

## 심화: 사용성 테스트 심화 가이드

### 테스트 유형별 비교
| 유형 | 방식 | 언제 |
|------|------|------|
| Moderated (진행자 있음) | 1:1 실시간 | 복잡한 플로우, 이유 탐구 |
| Unmoderated (비진행) | 혼자 수행 | 빠른 검증, 많은 샘플 |
| Remote (원격) | 화상/툴 | 지역 제약 없음 |
| In-person (대면) | 직접 | 풍부한 관찰, 표정/행동 |
| Guerrilla | 즉흥 테스트 | 빠른 피드백, 낮은 비용 |

### 과제(Task) 작성 가이드
**좋은 과제의 조건**
- 현실적인 시나리오 기반
- 기능명 언급 없음 (힌트 방지)
- 측정 가능한 완료 기준 존재
- 특정 경로를 유도하지 않음

**시나리오 템플릿**
```
"당신은 [역할]입니다. [상황]이 발생했습니다.
[목표]를 달성해 보세요."

예시: "당신은 팀 매니저입니다. 새로운 팀원이 합류했습니다.
팀원에게 프로젝트 접근 권한을 부여해 보세요."
```

### 관찰 포인트 체크리스트
- 어디서 멈추는가? (망설임 포인트)
- 어디를 먼저 클릭하는가? (첫 번째 클릭)
- 어디서 잘못된 경로로 가는가? (오류 경로)
- 어떤 요소를 무시하는가? (주목받지 못하는 요소)
- 어떤 표현/단어에 혼란을 보이는가?
- 완료 후 반응은? (만족/불만족)

### 정량 지표 수집
- **Task Success Rate**: 과제 완료 비율 (목표: 80%+)
- **Time on Task**: 과제 완료 시간 (기준값과 비교)
- **Error Rate**: 오류 발생 횟수
- **SUS(System Usability Scale)**: 10개 문항 설문, 68점 이상이 평균

### 결과 보고서 구조
1. 요약: 핵심 발견 3가지 + 권고 사항
2. 방법론: 참여자 프로필, 과제, 진행 방식
3. 발견 사항: 심각도별 분류 (Critical / Major / Minor)
4. 영상 클립: 각 발견의 증거 영상
5. 우선순위 매트릭스: 빈도 × 심각도
6. 권고 사항: 각 발견에 대한 디자인 개선안

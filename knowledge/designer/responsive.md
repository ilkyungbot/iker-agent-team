# 반응형 디자인 — 모든 화면에서 동작하는 UI

> 반응형은 화면을 축소하는 것이 아니다. 각 환경에 맞는 최적의 경험을 설계하는 것이다.

## 핵심 원칙

- **Mobile First**: 가장 작은 화면부터 설계하고 확장한다
- **콘텐츠 우선순위**: 화면이 작아질수록 핵심 콘텐츠만 남긴다
- **유동성**: 고정 너비보다 비율과 최대 너비 조합을 사용
- **브레이크포인트는 콘텐츠 기반**: 디바이스 크기보다 콘텐츠가 깨지는 지점 기준
- **터치와 포인터 동시 고려**: 같은 화면에서 마우스와 손가락 모두 지원할 수 있어야

## 판단 기준

**표준 브레이크포인트**
```
xs: 0~375px (소형 모바일)
sm: 376px~767px (모바일)
md: 768px~1023px (태블릿)
lg: 1024px~1279px (소형 데스크톱)
xl: 1280px~1535px (데스크톱)
2xl: 1536px~ (와이드 스크린)
```

**화면별 주요 변경 패턴**
- 내비게이션: 데스크톱 상단 바 → 모바일 햄버거/탭바
- 그리드: 12컬럼 → 8컬럼 → 4컬럼
- 카드 레이아웃: 가로 3열 → 2열 → 1열
- 폰트 크기: 소폭 감소 (Heading 1: 36px → 28px → 24px)

## 실전 예시

**Figma 반응형 설계 워크플로우**
1. 모바일 프레임(375px)에서 설계 시작
2. 컴포넌트에 Auto Layout + min/max width 적용
3. 태블릿(768px), 데스크톱(1280px) 프레임 추가
4. Constraints 설정으로 동작 방식 명세화
5. 각 브레이크포인트에서 overflow 케이스 확인

**텍스트 반응형 스케일 예시**
```
모바일: h1=24px, h2=20px, body=14px
태블릿: h1=28px, h2=22px, body=15px
데스크톱: h1=36px, h2=28px, body=16px
```

## 안티패턴

- 데스크톱 디자인 완료 후 모바일은 "알아서 줄여줘"
- 모바일에서 중요하지 않은 콘텐츠를 `display: none`으로만 숨김 (성능 문제)
- 모든 브레이크포인트에서 동일한 UX 패턴 강요
- 터치 타겟을 PC 기준으로만 설계 → 모바일에서 탭 실패
- 세로/가로 회전(landscape) 미고려

## 실전 팁

- 디자인 초반에 "이 화면의 모바일 버전을 어떻게 처리할 것인가?" 팀과 협의
- Figma Responsive Resize로 빠른 레이아웃 검증
- 실 기기(iPhone SE, iPad, 구형 안드로이드)에서 반드시 확인
- 개발 단계에서 브라우저 DevTools 반응형 뷰로 중간 검증 요청
- 이미지 크기 최적화: srcset과 sizes 어트리뷰트 명세 제공

## 심화: 반응형 설계 심화 가이드

### 유동적 타이포그래피 (Fluid Typography)
픽셀 기반 고정 크기 대신 viewport에 따라 선형으로 변화:
```css
/* clamp(최소값, 선호값, 최대값) */
font-size: clamp(1rem, 2.5vw, 2rem);
/* 모바일: 16px, 데스크톱: 최대 32px, 중간은 부드럽게 변화 */
```

### 컨테이너 쿼리 (Container Queries)
뷰포트가 아닌 부모 컨테이너 크기 기반 스타일:
```css
@container (min-width: 400px) {
  .card { flex-direction: row; }
}
```
- 컴포넌트가 배치된 컨텍스트에 따라 자동으로 레이아웃 변화
- 재사용 가능한 반응형 컴포넌트에 이상적

### 이미지 반응형 처리
```html
<img
  src="image-800.jpg"
  srcset="image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w"
  sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
  loading="lazy"
  alt="설명"
/>
```

### 반응형 테이블 처리 패턴
데이터 테이블은 모바일에서 가장 어려운 요소 중 하나:
1. **스크롤**: 테이블을 가로 스크롤 가능하게
2. **우선순위 컬럼**: 중요하지 않은 열 숨김
3. **카드 변환**: 각 행을 카드 형태로 전환
4. **Summary View**: 핵심 열만 표시 + 상세보기 CTA

### 반응형 네비게이션 전환
```
데스크톱(1024px+):
  상단 수평 네비게이션 바
  서브메뉴: hover 드롭다운

태블릿(768px~1023px):
  간소화된 상단 바 + 일부 항목 숨김
  오버플로우: "더 보기" 드롭다운

모바일(~767px):
  하단 탭바 (4~5개 핵심 항목)
  기타 항목: 햄버거 → 드로어
```

### 성능을 고려한 반응형
- 모바일에서 필요 없는 큰 이미지 로드 방지 (picture element)
- 데스크톱 전용 인터랙션 JS는 조건부 로드
- CSS-only 반응형 우선, JS는 마지막 수단
- Critical CSS 인라인으로 초기 렌더링 최적화

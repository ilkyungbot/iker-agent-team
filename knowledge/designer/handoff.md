# 핸드오프 — 디자인 의도가 개발로 전달되는 과정

> 핸드오프는 파일 전달이 아니다. 의도와 결정의 맥락을 전달하는 커뮤니케이션이다.

## 핵심 원칙

- **명세 완성도**: 개발자가 디자이너 없이 구현 가능한 수준의 명세
- **상태 완비**: 모든 인터랙티브 요소의 상태(hover, focus, disabled 등) 정의
- **토큰 연결**: 컬러, 간격, 타이포 모두 디자인 토큰으로 참조
- **의도 설명**: "왜 이렇게 결정했는가"를 주석이나 문서로 남긴다
- **지속적 협업**: 핸드오프는 끝이 아닌 시작. 구현 중 질문에 빠르게 응답

## 판단 기준

**핸드오프 완료 체크리스트**
- [ ] 모든 컴포넌트 state 설계 완료 (default/hover/focus/active/disabled/error)
- [ ] 반응형 동작 명세 (각 브레이크포인트별)
- [ ] 애니메이션/트랜지션 duration, easing 명시
- [ ] 접근성 명세 (aria-label, role, 키보드 동작)
- [ ] 에러/빈 상태/로딩 상태 디자인
- [ ] 에셋 export 설정 (SVG, PNG @1x/@2x/@3x)
- [ ] 디자인 토큰 연결 확인

## 실전 예시

**Figma 핸드오프 구조**
```
Page: [Handoff] Production
  ├── Cover (프로젝트 정보, 날짜, 담당자)
  ├── Design System Reference
  ├── Flows (전체 사용자 흐름)
  ├── Screens (화면별 명세)
  │   ├── [PC] 로그인
  │   ├── [Mobile] 로그인
  │   └── [States] 로그인 오류
  └── Assets (export 대상 에셋)
```

**주석 작성 예시**
- "버튼 비활성화 조건: 이메일·비밀번호 중 하나라도 빈 칸일 때"
- "카드 클릭 시 오른쪽에서 slide-in, 250ms ease-in-out"
- "최대 텍스트 2줄. 초과 시 말줄임 처리"

## 안티패턴

- 완성된 화면만 전달하고 상태/예외 케이스 누락
- 임의 컬러값 사용 (토큰 없이 #3B82F6 직접 사용)
- 개발자 질문에 "그냥 눈대중으로" 응답
- 핸드오프 후 디자인 변경사항 미공유
- 모든 명세를 노션에만 작성하고 Figma와 동기화 안 됨

## 실전 팁

- 핸드오프 전 개발자와 30분 walk-through 세션 필수
- Figma Dev Mode 활용: 개발자가 spacing, color, code snippet 직접 확인
- Notion + Figma 링크 양방향 연결로 문서 ↔ 디자인 탐색 가능
- "이 화면에서 가장 구현이 어려운 부분은?" 개발자에게 먼저 물어보기
- 구현 완료 후 QA는 Figma 원본과 비교 체크 (픽셀 퍼펙션 X, 의도 일치 O)

## 심화: 핸드오프 심화 가이드

### 핸드오프 유형별 전략
**기능 단위 핸드오프**
- 기능 범위 정의서 첨부
- 플로우 다이어그램 포함
- 관련 API 명세 링크

**디자인 시스템 컴포넌트 핸드오프**
- Props/Variants 전체 명세
- 사용 가이드라인 링크
- 접근성 명세 필수

**리디자인/마이그레이션 핸드오프**
- Before/After 비교 화면
- 변경 이유 문서화
- 마이그레이션 우선순위

### Figma Dev Mode 활용
- Inspect 패널: spacing, color, typography 자동 측정
- Code 패널: CSS/Swift/Kotlin 코드 스니펫 자동 생성
- Component 링크: Storybook/코드 저장소 연결
- Annotation 플러그인: 명세 주석 직접 추가

### 에셋 Export 명세
**아이콘**
- SVG 포맷, Outline stroke expanded
- currentColor로 컬러 변경 가능하게
- 24x24px 아트보드, 1px 여백 포함

**이미지**
- JPG: 사진, 배경 이미지 (quality 80%)
- PNG: 투명 배경, 레티나 (@1x, @2x)
- WebP: 웹 최적화 (AVIF도 고려)
- SVG: 로고, 일러스트, 아이콘

**스프라이트/Lottie**
- 애니메이션 JSON 파일 + 사용법 명세

### 개발 QA 체크리스트 (디자이너 기준)
구현 완료 후 디자이너가 확인해야 할 항목:
- [ ] 폰트 크기/굵기/행간 일치
- [ ] 컬러 토큰 올바르게 적용
- [ ] 간격(spacing) 시스템 준수
- [ ] Hover/Focus/Active 상태 구현
- [ ] 반응형 동작 (모바일/태블릿/데스크톱)
- [ ] 애니메이션 타이밍/이징 일치
- [ ] 에러/빈 상태 구현 완료
- [ ] 접근성 기본 사항 (포커스, alt 텍스트)

### 핸드오프 커뮤니케이션 팁
- Figma 링크 공유 시 직접 frame 선택하여 공유 (전체 파일 X)
- 주요 변경사항은 Slack/Linear 코멘트로 별도 공지
- "왜 이렇게 결정했는가"를 Figma 코멘트로 남기기
- 개발 중 발생하는 질문에 24시간 내 응답 목표

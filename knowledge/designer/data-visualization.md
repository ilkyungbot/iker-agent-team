# 데이터 시각화 — 숫자를 이해로 바꾸는 디자인

> 차트는 데이터를 보여주는 것이 아니라 인사이트를 전달하는 것이다.

## 핵심 원칙

- **목적 명확화**: "이 차트로 사용자가 무엇을 결정할 수 있어야 하는가?"
- **데이터 잉크 최소화**: 데이터를 나타내지 않는 요소는 모두 제거 대상
- **적합한 차트 유형**: 데이터 관계에 맞는 시각화 방법 선택
- **컨텍스트 제공**: 숫자만으론 부족. 비교 기준, 목표값, 추세 함께 표시
- **접근성**: 색상 외 패턴/레이블로 데이터 구분

## 판단 기준

**차트 유형 선택 가이드**
| 목적 | 권장 차트 |
|------|---------|
| 시간 변화 추세 | 라인 차트 |
| 항목 비교 | 막대 차트 (수평: 항목명 길 때) |
| 비율/구성 | 파이차트 (2~5개 항목), 스택 바 |
| 상관관계 | 산점도 |
| 분포 | 히스토그램, 박스플롯 |
| 지리 데이터 | 지도 시각화 |

## 실전 예시

**대시보드 계층 구조**
```
Level 1: KPI 카드 (핵심 지표 3~5개, 즉각 파악)
Level 2: 추세 차트 (시간 변화, 패턴 발견)
Level 3: 세부 분석 (드릴다운, 필터링)
```

**좋은 차트 제목 vs 나쁜 차트 제목**
- X "월별 매출 데이터" (설명만)
- O "3월 매출 전월 대비 23% 증가" (인사이트 전달)
- O "신규 가입자 수 — 2025년 추이" (명확한 맥락)

**색상 사용 원칙**
- 단일 데이터 시리즈: 단색 사용
- 여러 시리즈: 최대 7가지, 색맹 안전 팔레트 (ColorBrewer)
- 강조: 나머지를 회색으로, 하나만 브랜드 컬러

## 안티패턴

- 3D 차트 사용 (왜곡 발생, 가독성 저하)
- Y축이 0에서 시작하지 않는 막대 차트 (과장된 차이)
- 파이차트에 7개 이상 슬라이스
- 범례에만 의존, 직접 레이블 없음
- 모바일에서 데스크톱 차트 그대로 사용

## 실전 팁

- 차트 설계 전 "이 데이터로 내릴 결정은?"을 먼저 정의
- Figma Charts 플러그인 또는 실제 데이터로 프로토타입 검증
- 빈 상태, 데이터 없음, 로딩 상태 모두 설계
- 모바일 대시보드: 차트 대신 KPI 카드로 단순화 고려
- 대화형 차트(hover tooltip, 드릴다운) 설계 시 상태 명세 필수

## 심화: 데이터 시각화 심화 가이드

### 대시보드 설계 원칙
**정보 계층 설계**
1. Hero KPI: 가장 중요한 지표 1~3개. 즉각 파악 가능
2. 트렌드: 시간에 따른 변화. 맥락 제공
3. 분석: 세부 분해, 비교. 드릴다운
4. 원시 데이터: 테이블. 필요 시 export

**대시보드 레이아웃 패턴**
- 역 피라미드: 요약 → 세부 (위에서 아래)
- F 패턴: 중요 KPI를 왼쪽 상단에
- 필터를 상단에 배치하여 전체 영향

### 차트 설계 세부 지침
**라인 차트**
- 데이터 포인트 강조: 8개 이하면 dot 표시
- 여러 라인: 최대 5개, 직접 레이블로 범례 대체
- Y축 0 기준: 변화량이 중요하면 0 시작, 범위가 중요하면 자동

**막대 차트**
- 수직 막대: 시간 순서 비교에 적합
- 수평 막대: 항목명이 길 때, 순위 비교
- 그룹 막대: 최대 2~3개 그룹
- 적층 막대: 합계와 비율 동시 표현

**파이/도넛 차트**
- 2~5개 슬라이스만 권장
- 가장 큰 슬라이스를 12시 방향에
- 도넛 차트 중앙에 총합 또는 주요 지표 표시

### 실시간 데이터 시각화
- 새로운 데이터 추가 시 애니메이션으로 변화 강조
- 자동 새로고침 주기를 사용자가 설정 가능하도록
- 마지막 업데이트 시간 항상 표시
- 연결 끊김 상태 명확히 표시

### 빈 상태 처리
```
데이터가 없을 때:
  1. 이유 설명 ("아직 데이터가 없어요")
  2. 어떻게 채울 수 있는지 안내
  3. CTA 제공 ("첫 번째 데이터 추가하기")

필터 결과가 없을 때:
  1. 현재 적용된 필터 표시
  2. 필터 초기화 버튼 제공
  3. 검색 범위 조정 제안
```

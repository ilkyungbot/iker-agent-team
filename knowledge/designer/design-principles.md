# 디자인 원칙 — 좋은 디자인을 판단하는 기준

> 디자인은 아름다움이 아니라 문제 해결이다. 원칙은 일관된 판단의 근거가 된다.

## 핵심 원칙

- **목적 우선**: 모든 시각적 결정은 사용자 목표 달성을 돕는 방향이어야 한다
- **일관성**: 동일한 문제에는 동일한 해결책을 적용한다 (= 시스템 사고)
- **계층 구조**: 중요도에 따라 시각적 무게를 다르게 부여한다
- **여백은 요소다**: 공백은 비어있는 것이 아니라 숨을 쉬게 하는 디자인 요소
- **단순성**: 제거할 수 없는 것만 남긴다. 추가는 쉽고 제거는 어렵다

## 판단 기준

| 질문 | 좋은 디자인 | 나쁜 디자인 |
|------|------------|------------|
| 사용자가 다음에 뭘 해야 할지 아는가? | 명확한 CTA | 여러 선택지가 경쟁 |
| 3초 안에 페이지 목적을 파악할 수 있는가? | 계층이 뚜렷함 | 모든 요소가 동등한 무게 |
| 재방문 시 일관성이 느껴지는가? | 패턴이 반복됨 | 화면마다 다른 스타일 |

## 실전 예시

**계층 구조 적용**
- 헤드라인 → 서브헤드 → 본문 → 캡션 순으로 폰트 크기/굵기 감소
- 주요 CTA는 하나, 보조 액션은 ghost 또는 링크 버튼으로 처리

**여백 활용**
- 카드 내부 padding: 최소 16px, 관련 요소 간격 < 비관련 요소 간격
- 섹션 간격이 요소 간격보다 2배 이상 커야 그루핑이 명확해짐

## 안티패턴

- "더 추가하면 더 좋아 보일 것 같아" → 노이즈만 증가
- 모든 버튼을 Primary로 처리 → 시각적 계층 붕괴
- 일관성 없는 간격 값 (12px, 13px, 15px 혼용) → 디자인 시스템 불신
- 장식용 그래픽 남발 → 콘텐츠 가독성 저하

## 실전 팁

- 디자인 리뷰 전 "이 요소를 제거하면 사용자가 손해를 보는가?" 자문
- 레이아웃 판단 시 흑백으로 먼저 본다. 컬러는 계층을 숨긴다
- Gestalt 원칙 (근접성, 유사성, 연속성)을 의식적으로 활용
- 경쟁사 벤치마크보다 사용자 멘탈 모델을 먼저 파악
- "디자이너가 좋아하는 것"과 "사용자에게 필요한 것"을 구분하는 습관

## 심화: 주요 디자인 원칙 프레임워크

### Dieter Rams의 좋은 디자인 10원칙
1. 혁신적이다
2. 제품을 유용하게 만든다
3. 아름답다
4. 이해하기 쉽다
5. 눈에 띄지 않는다
6. 정직하다
7. 오래 지속된다
8. 세부적인 것도 철저하다
9. 환경 친화적이다
10. 가능한 적게 디자인한다

### Don Norman의 UX 7원칙
- **발견 가능성**: 가능한 액션이 눈에 보여야 함
- **피드백**: 시스템이 현재 상태를 알려야 함
- **개념 모델**: 사용자가 시스템 작동 방식을 이해해야 함
- **어포던스**: 요소의 생김새가 사용 방법을 암시
- **시그니파이어**: 행동이 가능하다는 신호 표시
- **매핑**: 컨트롤과 효과 간의 관계가 명확
- **제약**: 불필요한 선택지를 제한

### 실제 적용 체크리스트
- [ ] 이 화면의 핵심 목적은 한 문장으로 정의되는가?
- [ ] 사용자가 다음에 해야 할 행동이 명확한가?
- [ ] 불필요한 요소가 제거되었는가?
- [ ] 동일한 패턴이 서비스 전체에서 일관되게 사용되는가?
- [ ] 원칙을 벗어난 결정이 있다면 이유가 문서화되어 있는가?

### 원칙과 트레이드오프 관리
- 두 원칙이 충돌할 때는 "사용자 목표 달성"이 최우선
- 일관성 vs 맥락 최적화: 일관성이 기본, 맥락이 강할 때만 예외
- 단순성 vs 기능: 기능 추가는 복잡성을 높임. 신중하게
- 팀 내 원칙 불일치가 생기면 문서화된 의사결정 필요

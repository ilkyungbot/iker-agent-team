# 디자인 크리틱 — 더 나은 결과를 위한 구조화된 피드백

> 크리틱은 비판이 아니다. 디자인이 목표를 달성하는지 집단 지성으로 검증하는 과정이다.

## 핵심 원칙

- **목표 기준 피드백**: "나는 이게 싫어" → "이 선택이 사용자 목표 달성에 어떤 영향을 미치는가?"
- **발표자 역할 분리**: 발표자는 방어하지 않고 듣는다
- **구체성**: "좋아 보여요"는 피드백이 아니다. 무엇이 왜 좋은지를 말한다
- **심리적 안전감**: 피드백은 작업에 대한 것이지 사람에 대한 것이 아니다
- **액션 지향**: 크리틱은 다음 스텝으로 이어져야 한다

## 판단 기준

**피드백 품질 체크**
- 관찰 가능한 사실에 기반하는가?
- 사용자 관점에서 출발하는가?
- 구체적인 개선 방향을 제시하는가?
- 임의적 취향이 아닌 원칙에 근거하는가?

**크리틱 구조 (I Like / I Wish / What if)**
- "I Like": 효과적으로 작동하는 것
- "I Wish": 개선이 필요하다고 보이는 것
- "What if": 새로운 방향 제안

## 실전 예시

**좋은 피드백 vs 나쁜 피드백**
```
나쁨: "버튼 색이 별로예요"
좋음: "CTA 버튼이 배경과 대비가 낮아서 눈에 잘 안 띕니다.
       대비비를 높이거나 위치를 조정하면 클릭률이 개선될 것 같습니다"

나쁨: "저는 이런 레이아웃을 좋아하지 않아요"
좋음: "이 레이아웃에서 핵심 정보(가격)가 스크롤 아래에 있어
       사용자가 구매 결정 전에 이탈할 가능성이 있어 보입니다"
```

**크리틱 세션 운영**
1. 발표자: 디자인 목표와 제약 조건 설명 (5분)
2. 침묵 관찰: 참여자 개인 메모 (3~5분)
3. 피드백 공유: 라운드 로빈 또는 포스트잇
4. 논의 및 우선순위 결정
5. 발표자: 다음 스텝 정리

## 안티패턴

- 발표 중 즉각적 방어 ("그건 의도적으로...")
- 피드백 없이 "좋아 보여요"로 끝나는 세션
- 상급자 의견이 모든 피드백을 압도
- 크리틱 결과를 문서화하지 않음
- 피드백을 선택적으로만 반영하고 이유 공유 없음

## 실전 팁

- 크리틱 전 "오늘 이 세션에서 얻고 싶은 것"을 발표자가 명확히 제시
- 온라인 세션: FigJam에 피드백 스티커 직접 부착 방식으로 시각화
- 피드백 카테고리화: 기능/시각/접근성/비즈니스 로 분류
- 크리틱 후 72시간 내에 반영 결과 공유 (신뢰 구축)
- 주 1회 팀 크리틱 정례화로 디자인 품질 지속 향상

## 심화: 디자인 크리틱 심화 가이드

### 크리틱 세션 유형
**자기 크리틱 (Self-Critique)**
- 공유 전 자신의 디자인을 먼저 비판적으로 검토
- 체크리스트: 목표 달성? 일관성? 접근성? 사용성?
- 24시간 후 다시 보는 "냉각 리뷰" 효과적

**팀 크리틱 (Team Critique)**
- 주 1회 정례화, 30~60분
- 2~3개 작업물만 다루기 (많으면 표면적 피드백)
- 발표자 외 모든 팀원이 피드백 의무

**전문가 리뷰 (Expert Review)**
- UX 전문가가 휴리스틱 평가
- Nielsen의 10가지 휴리스틱 기준 체계적 평가
- 사용자 테스트 전 빠른 문제 발견에 유용

### 피드백 심각도 분류
```
P0 (Critical): 사용자가 핵심 목표를 완수할 수 없음
P1 (Major): 사용성에 심각한 영향, 수정 필요
P2 (Minor): 개선 가능하지만 차선책 존재
P3 (Nice-to-have): 개선하면 좋지만 우선순위 낮음
Observation: 정보 공유 목적, 액션 없음
```

### 원격 크리틱 운영
**비동기 크리틱 (Async)**
- Figma 코멘트로 피드백 작성
- 24~48시간 내 응답 기한 설정
- 장점: 시차 팀, 더 깊은 사고 가능
- Loom 영상 피드백으로 의도 명확히 전달

**실시간 원격 크리틱**
- Zoom + FigJam 조합
- 화면 공유 + 실시간 스티커 붙이기
- 마이크 끄고 침묵 관찰 시간 확보

### 피드백 받는 방법 (발표자)
1. 방어적 반응 억제: "그런데..." → 메모만 하고 끄덕
2. 명확화 질문: "어떤 부분이 혼란스러웠나요?"
3. 모든 피드백 기록: 동의 안 해도 일단 적기
4. 우선순위 결정: 모든 피드백을 다 따를 필요 없음
5. 반영 결과 공유: 수용/미수용 이유와 함께

### 크리틱 문화 구축
- 안전한 공간 선언: 피드백은 작업에 대한 것, 사람에 대한 것이 아님
- 궁정파 방지: "좋아요"만 하면 다음 라운드에 질문 의무
- 긍정도 포함: 잘 된 것을 명시적으로 인정
- 정기성: 불규칙한 크리틱보다 짧고 자주가 효과적

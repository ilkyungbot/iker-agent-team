# 폼 디자인 — 사용자의 입력 부담을 최소화하는 설계

> 폼은 사용자와 시스템 간의 계약이다. 명확할수록 오류가 줄고 완료율이 높아진다.

## 핵심 원칙

- **필요한 것만 묻기**: 모든 필드는 "이게 정말 필요한가?"를 정당화해야 한다
- **레이블 항상 표시**: placeholder는 레이블을 대체할 수 없다
- **실시간 검증**: 제출 후가 아닌 입력 중 또는 포커스 아웃 시 검증
- **오류 메시지 구체화**: "오류가 발생했습니다" → "이메일 형식이 올바르지 않습니다"
- **진행 상황 표시**: 긴 폼은 단계를 나누고 진행률을 보여준다

## 판단 기준

**레이블 위치 결정**
- Top label (권장): 읽기 쉽고, 긴 레이블도 잘림 없이 표시
- Left label: 데이터 입력 위주의 관리 도구에서 공간 효율
- Floating label: 심미적이나 인식 혼란 가능성. 신중하게 사용

**입력 필드 크기**
- 모바일: 최소 44px 높이
- 데스크톱: 36~44px 높이
- 너비: 예상 입력 길이에 맞게 (전화번호 < 주소 < 메모)

## 실전 예시

**오류 메시지 설계 원칙**
```
나쁨: "유효하지 않은 입력입니다"
좋음: "이메일 주소에 @가 포함되어야 합니다"
좋음: "비밀번호는 8자 이상, 영문+숫자 조합이어야 합니다"
```

**폼 완료율 높이는 패턴**
- 선택 필드는 명확히 "(선택)" 표기
- 비밀번호 보기/숨기기 토글 제공
- 자동완성 및 이전 입력값 제안
- 긴 드롭다운 대신 검색 가능한 셀렉트 사용
- 주소: 우편번호 API로 자동 완성

## 안티패턴

- 제출 버튼 클릭 후에야 오류 표시 (모든 오류가 한꺼번에)
- 빨간 테두리만으로 오류 표시 (색맹 접근성 문제)
- 모든 필드를 필수로 설정
- 입력 완료 전에 오류 표시 (입력 중인데 "이메일 형식 오류")
- 비밀번호 확인 필드 없이 비밀번호 입력만 요청

## 실전 팁

- 폼 필드 수와 전환율은 반비례. 필드 1개 줄이면 전환율 증가
- 인라인 검증 타이밍: focus-out 또는 일정 시간 입력 정지 후
- 모바일에서 키보드 유형을 input type으로 최적화 (tel, email, number)
- 폼 제출 후 성공 상태를 명확히 표시하고 다음 단계 안내
- 에러 상태에서 성공적으로 수정되면 즉시 오류 메시지 제거

## 심화: 폼 설계 심화 가이드

### 폼 레이아웃 패턴
**단일 컬럼 레이아웃 (권장)**
- 시선이 위에서 아래로 자연스럽게 흐름
- 모바일과 데스크톱 모두 동작
- 입력 완료 경로가 명확

**다중 컬럼 레이아웃 (제한적 사용)**
- 관련 있는 짧은 필드 쌍에만 사용 (이름/성, 시/도)
- 랜덤하게 나란히 배치하면 완료 경로 혼란

### 입력 유형별 최적 컴포넌트
| 데이터 유형 | 권장 컴포넌트 | 비권장 |
|-----------|-------------|--------|
| 단일 선택 (2~4개) | Radio Button | Select |
| 단일 선택 (5개+) | Select / Combobox | Radio |
| 다중 선택 (2~5개) | Checkbox | Multi-select |
| 날짜 | Date Picker | Text Input |
| 금액/숫자 | Number Input + format | Text |
| 긴 텍스트 | Textarea | Input |
| 토글 설정 | Toggle Switch | Checkbox |

### 폼 유효성 검사 타이밍
```
입력 시작 전: 검사 안 함 (입력을 방해하지 않도록)
입력 중: 형식이 맞아가는 경우 긍정 피드백 가능
포커스 아웃: 빈칸/형식 오류 검사 시작
제출 시: 전체 검사 + 첫 번째 오류로 스크롤 + 포커스
```

### 다단계 폼 설계
- 진행률 표시: "단계 2/4" 또는 스텝 인디케이터
- 각 단계는 논리적 그룹 (개인정보 / 주소 / 결제)
- 이전 단계로 돌아가도 입력값 유지
- 저장 및 나중에 계속 기능 고려 (긴 폼)
- 최종 제출 전 요약 확인 화면 제공

### 자동완성 최적화 명세
```html
이름: autocomplete="name"
이메일: autocomplete="email"
전화: autocomplete="tel"
주소: autocomplete="street-address"
우편번호: autocomplete="postal-code"
신용카드: autocomplete="cc-number"
비밀번호 (로그인): autocomplete="current-password"
비밀번호 (회원가입): autocomplete="new-password"
```

### 오류 복구 패턴
- 오류 필드로 자동 스크롤 + 포커스 이동
- 오류 목록을 상단에 요약 (접근성)
- 올바르게 수정된 필드는 즉시 성공 표시
- 서버 오류는 구체적 안내 + 재시도 버튼

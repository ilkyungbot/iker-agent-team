# UX 라이팅 — 인터페이스 속 언어의 디자인

> UI 텍스트는 기능 설명이 아니다. 사용자를 다음 행동으로 안내하는 대화다.

## 핵심 원칙

- **명확성**: 이해하기 위해 두 번 읽을 필요가 없어야 한다
- **간결성**: 목적 달성에 필요한 최소한의 단어만 사용
- **사람처럼**: 기계적이거나 법적 언어 대신 자연스러운 대화체
- **능동태**: "데이터가 저장됩니다" → "저장했어요"
- **일관성**: 같은 개념은 항상 같은 단어로 표현

## 판단 기준

**버튼 레이블 원칙**
- 동사+목적어: "파일 업로드", "결제하기", "초대 보내기"
- "확인", "OK", "예"는 가능한 피하기 → 무엇을 확인하는지 명시
- 위험한 액션: "삭제", "탈퇴" 등 행동 결과를 명확하게

**오류 메시지 3요소**
1. 무엇이 잘못됐는가 (문제 설명)
2. 왜 발생했는가 (원인, 간결하게)
3. 어떻게 해결하는가 (해결 방법)

## 실전 예시

**Empty State 텍스트**
```
나쁨: "데이터 없음"
좋음: 
  제목: "아직 프로젝트가 없어요"
  설명: "새 프로젝트를 만들면 여기에 표시됩니다"
  CTA: "첫 프로젝트 만들기"
```

**로딩/처리 중 텍스트**
```
나쁨: "Loading..." / "처리 중..."
좋음: "파일을 업로드하고 있어요" / "결제를 처리하고 있어요"
```

**성공 메시지**
```
나쁨: "작업이 완료되었습니다"
좋음: "보고서가 저장됐어요" / "초대 이메일을 보냈어요"
```

## 안티패턴

- "클릭하세요" (터치 기기에선 부적절)
- 부정적 언어 남발: "접근 불가", "오류", "실패"
- 시스템 에러 코드 그대로 노출: "Error 500"
- 불필요한 경고 ("이 작업은 되돌릴 수 없습니다" — 모든 버튼에)
- 지나치게 친근한 말투가 진지한 상황에 부적절

## 실전 팁

- Tone of Voice 가이드를 팀 공유 문서로 유지
- UX 라이터가 없는 팀: PM or 디자이너가 초안 작성 → 피드백 사이클
- A/B 테스트로 CTA 문구 효과 측정 (전환율 차이 발생)
- 플레이스홀더는 예시 텍스트로: "예: hong@company.com"
- 한국어 존댓말/반말 정책을 서비스 톤에 맞게 사전 결정

## 심화: UX 라이팅 심화 가이드

### Tone of Voice 정의
**4가지 차원으로 정의**
1. 재미있는 ↔ 진지한
2. 격식 있는 ↔ 격식 없는
3. 공손한 ↔ 솔직한
4. 열정적인 ↔ 차분한

**예시: B2B SaaS 서비스**
- 전문적이고 명확하지만 딱딱하지 않게
- 능동태, 짧은 문장
- 전문 용어는 사용하되 설명 병용
- 존댓말 (합쇼체/해요체 중 선택 일관)

### 콘텐츠 유형별 가이드
**온보딩 문구**
- 가치를 먼저, 방법은 나중에
- "설정을 완료하세요" X → "5분으로 팀 협업 준비 완료" O
- 진행률과 함께 동기 부여

**빈 상태 문구**
```
제목: [무엇이 없는지] + [긍정적 표현]
설명: [어떻게 시작할 수 있는지]
CTA: [첫 번째 액션]

예시:
제목: "아직 보고서가 없어요"
설명: "분석 결과를 보고서로 저장하면 여기에 모입니다"
CTA: "첫 보고서 만들기"
```

**오류 메시지 톤**
- 시스템 탓 (X): "처리할 수 없는 요청입니다"
- 중립 (O): "저장하지 못했어요. 잠시 후 다시 시도해 주세요"
- 해결책 제시: "문제가 계속되면 support@company.com으로 문의해 주세요"

### 번역/현지화 고려
- 한국어 텍스트는 영어 대비 1.3~1.5배 길어짐 → UI 공간 여유 확보
- 명사형 종결어미(~됩니다) vs 동사형(~해요) 일관성 유지
- 존칭 수준: 타깃 사용자와 서비스 톤에 맞게 결정
- 날짜 형식: YYYY.MM.DD (한국), MM/DD/YYYY (미국)

### UX 라이팅 리뷰 프로세스
1. 원고 작성 (디자이너 or UX 라이터)
2. 사용성 관점 리뷰 (명확성, 간결성)
3. 브랜드 관점 리뷰 (톤 일관성)
4. 법무/컴플라이언스 리뷰 (필요 시)
5. 원어민/현지인 리뷰 (다국어 출시 시)
6. A/B 테스트로 효과 측정

### 좋은 마이크로카피 예시
- 로딩: "거의 다 됐어요..." (아무것도 없는 것보다 나음)
- 삭제 확인: "정말 삭제할까요? 되돌릴 수 없어요" + [취소] [삭제하기]
- 비밀번호 규칙: 입력 중 실시간으로 충족 여부 체크 표시
- 결제 완료: "주문이 접수됐어요! 영수증을 이메일로 보내드렸어요"

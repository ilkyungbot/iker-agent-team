# 컬러 — 감정과 기능을 동시에 전달하는 도구

> 색상은 장식이 아니라 커뮤니케이션이다. 의미 없이 쓰인 색은 노이즈다.

## 핵심 원칙

- **시맨틱 역할 정의**: Primary, Secondary, Semantic(Success/Warning/Error/Info), Neutral
- **접근성 필수**: WCAG AA 기준 최소 4.5:1 대비비 (일반 텍스트)
- **60-30-10 법칙**: 주색 60%, 보조색 30%, 강조색 10%
- **다크모드 고려**: 컬러 정의 시 라이트/다크 양쪽을 함께 설계
- **문화적 맥락**: 색상의 의미는 문화권마다 다르다 (빨강: 경고 vs 행운)

## 판단 기준

| 컬러 역할 | 사용 목적 | 주의사항 |
|----------|----------|---------|
| Primary | 핵심 액션, 브랜드 표현 | 남용 시 강조 효과 소멸 |
| Secondary | 보조 액션, 배경 강조 | Primary와 충분한 차별화 |
| Success (Green) | 완료, 확인, 긍정 피드백 | 색맹 사용자 위해 아이콘 병용 |
| Warning (Yellow) | 주의 필요, 임박한 문제 | 배경에 사용 시 텍스트 대비 체크 |
| Error (Red) | 오류, 위험, 삭제 | 공포감 유발 최소화, 해결책과 함께 제시 |
| Neutral | 텍스트, 배경, 구분선 | 가장 많이 쓰이는 컬러, 스케일 정밀하게 |

## 실전 예시

**컬러 스케일 구조**
```
Primary
  50: 가장 연한 (배경, hover)
  100~400: 중간 톤
  500: Base color (메인 사용)
  600~700: 어두운 톤 (hover state, 다크모드)
  800~900: 가장 진한 (텍스트 on light bg)
```

**Semantic 컬러 예시**
- `color-text-primary`: #111827 (light) / #F9FAFB (dark)
- `color-bg-surface`: #FFFFFF (light) / #1F2937 (dark)
- `color-border-default`: #E5E7EB (light) / #374151 (dark)

## 안티패턴

- 브랜드 컬러를 배경색으로 넓게 사용 → 눈의 피로
- Error 상태를 색상만으로 표현 → 색맹 사용자 배제
- 임의의 hex 값 사용 (시스템 밖의 컬러)
- 너무 많은 강조색으로 시선 분산
- 다크모드를 단순 색상 반전으로 처리 → 대비비 역전 문제

## 실전 팁

- Figma에서 컬러 스타일/변수 미정의 상태로 컴포넌트 제작 금지
- 디자인 완료 후 Contrast Checker (plugins: Able, Stark)로 전체 점검
- 색맹 시뮬레이션 (Figma: Color Blind 플러그인) 필수 확인
- 컬러 네이밍은 색상명(blue)이 아닌 역할명(primary)으로
- 인쇄·외부 채널 납품 시 RGB/HEX → CMYK 변환 결과 확인

## 심화: 컬러 시스템 구축 가이드

### 컬러 팔레트 생성 방법
**HSL 기반 스케일 생성**
- Hue: 고정 (브랜드 컬러의 색상)
- Saturation: 50(연함) → 100(진함) 단계적 조정
- Lightness: 95(가장 연함) → 5(가장 진함) 10 단위 조정

**도구 추천**
- Tailwind CSS Color Generator
- Radix UI Colors (접근성 기반 팔레트)
- Colorbox by Lyft
- Realtime Colors

### 컬러 접근성 빠른 체크
```
흰 배경(#FFFFFF) 위 텍스트:
  AA 통과: #767676 이상 어두운 색
  AAA 통과: #595959 이상 어두운 색

검정 배경(#000000) 위 텍스트:
  AA 통과: #767676 이상 밝은 색
  AAA 통과: #9A9A9A 이상 밝은 색
```

### 브랜드 컬러 선택 원칙
- 경쟁사와 차별화 (카테고리 내 컬러 맵 먼저 파악)
- 서비스 톤에 부합: 금융=파랑(신뢰), 헬스=초록(생명), 엔터=보라(창의)
- 확장성: 다크모드, 인쇄, 외부 채널에서도 동작하는가
- 문화적 맥락: 글로벌 서비스라면 지역별 색상 의미 조사

### 다크모드 컬러 전환 원칙
- 배경: 흰색 → 어두운 회색 (완전 검정은 피함)
- 텍스트: 검정 → 밝은 회색 (완전 흰색은 눈 피로)
- Primary: 채도 낮추고 명도 높임 (진동 방지)
- 그림자: 투명도 기반 어두운 그림자 → 밝은 테두리로 대체

### 그라데이션 사용 기준
- 동일 색조 내 명도 변화: 안전하고 자연스러움
- 보색 그라데이션: 중간 지점이 탁해질 수 있어 주의
- 브랜드 그라데이션은 일러스트/배경에만 제한적 사용
- 텍스트에 그라데이션 적용 시 접근성 체크 어려움

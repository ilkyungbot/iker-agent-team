# 프로토타이핑 — 만들기 전에 검증하는 기술

> 프로토타입은 완성품이 아니다. 가장 빠르게 배우기 위한 도구다.

## 핵심 원칙

- **목적 중심 충실도**: 테스트 목적에 맞는 충실도만 사용. 불필요한 디테일은 낭비
- **빠른 실패**: 비싼 개발보다 싼 프로토타입으로 오류를 먼저 발견
- **실제 콘텐츠**: 최대한 실제 데이터와 텍스트로 프로토타이핑
- **인터랙션 명세**: 프로토타입이 핸드오프 명세를 대체하지 않는다
- **버전 관리**: 테스트할 때마다 버전 명시, 피드백과 버전 연결

## 판단 기준

**충실도(Fidelity) 선택 기준**
| 목적 | 충실도 | 도구 |
|------|--------|------|
| 아이디어 탐색 | 저 (Lo-Fi) | 종이, Balsamiq |
| 흐름 검증 | 중 (Mid-Fi) | Figma 와이어프레임 |
| 사용성 테스트 | 중-고 | Figma Interactive |
| 인터랙션 명세 | 고 (Hi-Fi) | Figma + Prototype |
| 개발 검증 | 최고 (코드) | Framer, React 스냅샷 |

## 실전 예시

**Figma 프로토타입 워크플로우**
1. 핵심 플로우만 연결 (온보딩, 결제, 핵심 기능)
2. Overlays로 모달/드로어 구현
3. Component Variants로 상태 전환 표현
4. Prototype Starting Point 설정
5. Present 모드로 실기기 테스트

**프로토타입 테스트 전 체크리스트**
- [ ] 시작 화면이 설정되어 있는가?
- [ ] 핵심 플로우의 모든 클릭 경로가 연결되어 있는가?
- [ ] 에러/빈 상태 화면도 포함되어 있는가?
- [ ] 실제 기기 해상도로 확인했는가?

## 안티패턴

- 프로토타입 완성도에 집착하여 테스트 지연
- 모든 버튼에 클릭 이벤트 연결하려는 욕심
- 프로토타입과 최종 디자인 싱크가 맞지 않음
- 테스트 피드백을 반영하지 않고 다음 테스트로
- 개발자에게 프로토타입만 넘기고 명세 없음

## 실전 팁

- "이 프로토타입으로 무엇을 검증할 것인가?" 한 문장으로 먼저 정의
- Figma Mirror 앱으로 실기기 테스트 (별도 빌드 불필요)
- Loom으로 프로토타입 시연 영상 제작 → 원격 피드백에 효과적
- 프로토타입 링크 + 테스트 목적을 함께 공유
- 결정이 끝난 프로토타입은 archive 처리. 현재 버전만 노출

## 심화: 프로토타입 유형별 가이드

### 종이 프로토타입 활용
- 언제: 아이디어 초기 탐색, 팀 브레인스토밍
- 방법: A4 또는 포스트잇에 스케치 → 직접 페이지 넘기기
- 장점: 빠름(30분), 변경 부담 없음, 아이디어에 집착하지 않게 됨
- 주의: 테스트 참여자가 완성도에 신경 쓰지 않도록 안내

### Figma Interactive 프로토타입 팁
- **Component Swap**: 버튼 상태 전환은 Variant 컴포넌트로
- **Overlay**: 모달, 토스트, 드로어는 Overlay 연결
- **Scroll Behavior**: 고정 헤더 설정, 스크롤 컨테이너 지정
- **Variables**: Figma Variables로 조건부 로직 구현 (로그인 상태 등)
- **Flow**: 여러 플로우를 Starting Frame으로 구분 관리

### 프로토타입 충실도 결정 기준
```
질문: 이 테스트에서 무엇이 가장 중요한가?

흐름 이해가 중요 → 와이어프레임 수준 (Mid-Fi)
비주얼 반응 측정 → 고충실도 (Hi-Fi)
인터랙션 디테일 검증 → Framer/코드 프로토타입
빠른 아이디어 검증 → 종이/저충실도
```

### 원격 프로토타입 테스트
- **Maze**: 클릭 경로, 히트맵, 성공률 자동 측정
- **UserTesting**: 실시간 화면 공유 + 음성 녹화
- **Figma Share Link**: 간단한 흐름 확인용 공유
- **Loom 시연 영상**: 비동기 피드백 수집

### 프로토타입 버전 관리
- 버전 명: v0.1 (탐색) → v0.5 (검증) → v1.0 (핸드오프)
- 각 버전에 테스트 목적과 주요 변경사항 메모
- 이전 버전은 별도 페이지로 아카이브 (삭제 금지)
- 핸드오프용 최종 버전은 [FINAL] 태그 표시

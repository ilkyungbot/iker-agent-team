# 레이아웃 — 공간을 통해 구조를 만드는 기술

> 레이아웃은 콘텐츠를 배치하는 것이 아니라, 사용자의 시선을 안내하는 것이다.

## 핵심 원칙

- **그리드 시스템**: 임의 배치 대신 컬럼/거터/마진 기반 그리드 사용
- **8pt 간격 시스템**: 4, 8, 12, 16, 24, 32, 48, 64px 등 8의 배수 사용
- **시각적 계층**: Z축(elevation)과 XY축(position) 모두 계층 표현에 활용
- **프록시미티 (근접성)**: 관련 요소는 가깝게, 비관련 요소는 멀게
- **정렬**: 왼쪽 정렬이 기본. 중앙 정렬은 제목·히어로 등 제한적 사용

## 판단 기준

| 화면 너비 | 그리드 컬럼 | 거터 | 마진 |
|---------|-----------|------|------|
| ~375px (모바일) | 4컬럼 | 16px | 16px |
| ~768px (태블릿) | 8컬럼 | 24px | 24px |
| ~1280px (데스크톱) | 12컬럼 | 24px | 40px |
| 1280px~ (와이드) | 12컬럼 | 32px | Auto (max-width) |

## 실전 예시

**컨텐츠 최대 너비 기준**
- 텍스트 블록: 680px (가독성 최적)
- 일반 콘텐츠 영역: 1024~1200px
- 대시보드/데이터: 1440px까지 허용

**공간 계층 패턴**
```
Page margin (40px)
  Section gap (64px)
    Component gap (32px)
      Element gap (16px)
        Inner padding (12px~16px)
```

## 안티패턴

- 픽셀 퍼펙션에 집착하여 홀수 px 간격 남발
- 텍스트 줄 길이 제한 없이 full-width 배치 → 가독성 저하
- 모든 섹션에 배경색 변화를 주어 레이아웃 분절감 심화
- 좌우 마진 없이 에지-투-에지 콘텐츠 배치
- 그리드를 무시한 자유 배치 후 "눈대중으로 맞춤"

## 실전 팁

- Figma Layout Grid를 항상 켜두고 작업. 그리드에서 벗어나면 의도적으로
- Auto Layout 적극 활용: 하드코딩된 여백 대신 gap/padding으로 제어
- 화면 공간이 부족할 때 콘텐츠 제거 검토 전에 우선순위 재조정
- 실제 콘텐츠 양이 예상과 다를 때를 대비해 overflow 케이스 설계
- 레이아웃 결정은 와이어프레임 단계에서 확정. 비주얼 단계에서 바꾸면 비용 증가

## 심화: 레이아웃 패턴 가이드

### 자주 쓰이는 레이아웃 패턴
**Holy Grail Layout**
- 헤더 / (사이드바 + 콘텐츠 + 사이드바) / 푸터
- 관리 도구, 문서 사이트에 적합

**Card Grid**
- 동일한 크기의 카드를 그리드로 배치
- 제품 목록, 대시보드 위젯, 콘텐츠 피드

**Bento Box**
- 크기가 다른 카드를 비대칭적으로 배치
- 랜딩 페이지, 포트폴리오, 데이터 하이라이트

**Split Screen**
- 화면을 두 영역으로 분할
- 비교, 로그인/회원가입, 마스터-디테일 패턴

### Z-패턴과 F-패턴
**Z-패턴**: 랜딩 페이지, 마케팅 페이지
- 눈이 Z 형태로 움직임: 상단 좌 → 우 → 좌 → 우하단
- 핵심 요소를 Z의 꼭짓점에 배치

**F-패턴**: 텍스트 중심 페이지, 뉴스, 목록
- 첫 두 줄을 왼쪽에서 오른쪽으로 읽고 이후 왼쪽만 스캔
- 중요한 정보를 왼쪽 상단에 집중 배치

### Auto Layout 활용 원칙 (Figma)
- 모든 컴포넌트는 Auto Layout으로 제작
- Hug Contents vs Fixed Size 구분하여 적용
- Gap, Padding을 spacing 토큰으로 연결
- Resizing 옵션: Fill container / Hug contents / Fixed
- Absolute Position은 오버레이 요소에만 제한적으로 사용

### 반응형 레이아웃 결정 트리
콘텐츠가 넘칠 때:
1. 폰트 크기 축소 (최소 12px까지)
2. 패딩/마진 축소
3. 컬럼 수 줄이기
4. 스택 레이아웃으로 전환
5. 콘텐츠 숨김 (낮은 우선순위 항목)

# 네비게이션 — 사용자가 항상 자신의 위치를 아는 구조

> 좋은 네비게이션은 "나는 어디에 있는가", "어디로 갈 수 있는가"를 항상 알려준다.

## 핵심 원칙

- **위치 인식**: 현재 위치를 명확히 표시 (active state, breadcrumb)
- **예측 가능성**: 같은 라벨은 항상 같은 목적지로
- **일관된 위치**: 네비게이션은 모든 화면에서 동일한 위치에
- **계층 반영**: 현재 위치가 전체 구조에서 어디인지 알 수 있어야
- **이탈 방지**: 사용자가 깊은 뎁스에서도 쉽게 돌아올 수 있어야

## 판단 기준

**네비게이션 패턴 선택 기준**
| 패턴 | 적합한 상황 | 주의사항 |
|------|-----------|---------|
| 상단 Nav Bar | 3~7개 주요 섹션 | 모바일에서 복잡해짐 |
| 사이드바 | 많은 메뉴 아이템, 관리도구 | 모바일 공간 부족 |
| 탭 바 (하단) | 모바일 4~5개 섹션 | 5개 초과 금지 |
| 햄버거 메뉴 | 보조 메뉴, 전체 메뉴 | 발견 가능성 낮음 |
| Breadcrumb | 깊은 계층 구조 | 모바일에서 공간 부족 |

## 실전 예시

**Breadcrumb 설계 원칙**
```
홈 > 상품 > 전자제품 > 노트북 > 맥북 프로 14인치
     ↑클릭 가능  ↑클릭 가능   ↑현재 위치 (비활성)
```

**모바일 네비게이션 전환 패턴**
- 데스크톱 상단 탭 → 모바일 하단 탭바 (아이콘 + 레이블)
- 데스크톱 사이드바 → 모바일 오버레이 드로어
- 서브 네비게이션 → 수평 스크롤 가능한 칩/탭

**Active state 명세**
- 배경색 변화 + 텍스트 굵기 변화 (두 가지 이상 신호 조합)
- 아이콘 filled vs outline으로 구분 (Material Design 방식)

## 안티패턴

- Active state 없이 현재 위치 불분명
- 메뉴 라벨과 실제 페이지 내용 불일치
- 네비게이션이 화면마다 다른 위치/구성
- 뒤로가기 버튼 없는 깊은 뎁스 화면
- 모바일에서 hover-only 드롭다운 메뉴

## 실전 팁

- 네비게이션은 디자인 시스템에서 가장 먼저 확정해야 할 요소
- 사용자 테스트에서 "지금 어디에 있는 것 같으세요?" 주기적으로 질문
- 검색 기능을 항상 네비게이션 근처에 배치
- 5단계 이상 뎁스가 필요하다면 IA 재구조화를 먼저 검토
- URL 구조와 네비게이션 계층이 일치해야 SEO + UX 모두 좋아짐

## 심화: 네비게이션 설계 심화 가이드

### 네비게이션 상태 설계
**모든 네비게이션 아이템의 필요 상태**
- Default: 기본 표시 상태
- Hover: 마우스 올렸을 때 (모바일 불필요)
- Active/Current: 현재 위치 표시
- Focus: 키보드 포커스 상태
- Notification: 뱃지/카운트 표시
- Collapsed: 사이드바 접힌 상태

### 메가 메뉴 설계 원칙
- 컬럼 수: 최대 4컬럼
- 아이템 수: 컬럼당 최대 8개
- 카테고리 제목을 명확히 구분
- 시각적 계층으로 주요/보조 링크 구분
- hover로 열리는 경우 300ms 딜레이로 의도치 않은 열림 방지

### 모바일 드로어 네비게이션
```
열기: 햄버거 버튼 탭 → 왼쪽에서 오버레이로 슬라이드 인
닫기: X 버튼 or 배경 탭 or 오른쪽으로 스와이프
애니메이션: 300ms ease-in-out
너비: 화면의 80% 또는 최대 320px
오버레이 배경: rgba(0,0,0,0.5)
포커스 트랩: 열린 동안 포커스가 드로어 밖으로 나가지 않도록
```

### 탭 바 설계 규칙
- 아이템 수: 3~5개 (2개면 탭 불필요, 6개+ 하면 인식 어려움)
- 구성: 아이콘 + 레이블 (아이콘만은 인식률 낮음)
- 현재 탭: filled 아이콘 + 색상 변화
- 뱃지: 숫자(99+로 제한) 또는 dot
- 탭 전환 시 각 탭의 스크롤 위치 유지

### URL과 네비게이션 일치
```
좋은 URL 구조 예시:
  /dashboard (대시보드)
  /projects (프로젝트 목록)
  /projects/123 (프로젝트 상세)
  /projects/123/settings (프로젝트 설정)
  
나쁜 URL 구조:
  /page?id=123&view=detail&tab=settings
```
- URL이 계층을 반영하면 breadcrumb 자동 생성 가능
- Deep link 지원으로 특정 화면 직접 접근 가능
- 브라우저 뒤로가기와 앱 내 뒤로가기 일관성 유지

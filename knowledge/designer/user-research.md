# 사용자 리서치 — 가정이 아닌 증거로 디자인하기

> 사용자는 당신이 아니다. 리서치는 자신의 편견을 깨는 과정이다.

## 핵심 원칙

- **공감적 중립**: 원하는 답을 유도하지 않는다. 사실을 발견하는 것이 목적
- **행동 > 말**: 사용자가 말하는 것보다 실제로 하는 것을 관찰하라
- **맥락 중요**: 실제 환경에서 사용하는 방식이 연구실 결과와 다르다
- **작은 샘플로도 충분**: 정성 리서치는 5~8명으로 주요 패턴 80%를 발견
- **지속적 리서치**: 출시 전 일회성이 아닌 지속적 사이클로 운영

## 판단 기준

| 리서치 방법 | 적합한 질문 | 샘플 수 |
|-----------|-----------|---------|
| 심층 인터뷰 | "왜 이렇게 행동하는가?" | 5~12명 |
| 사용성 테스트 | "이 UI를 사용할 수 있는가?" | 5~8명 |
| 설문조사 | "얼마나 많은 사람이 이렇게 생각하는가?" | 100명+ |
| 현장 관찰 | "실제 사용 맥락은 어떠한가?" | 4~8 세션 |
| A/B 테스트 | "A와 B 중 어느 것이 더 나은가?" | 통계적 유의성 필요 |

## 실전 예시

**인터뷰 좋은 질문 vs 나쁜 질문**
- X "이 기능이 유용하다고 생각하시나요?" (유도)
- O "지난주에 비슷한 작업을 어떻게 처리하셨나요?" (행동 탐색)
- X "불편한 점이 있었나요?" (부정적 답을 유도)
- O "이 작업을 할 때 가장 많은 시간이 걸리는 부분이 어디인가요?" (구체적)

**인사이트 도출 프로세스**
1. 원시 데이터 수집 (녹음, 노트, 스크린 녹화)
2. Affinity Diagram: 포스트잇으로 패턴 클러스터링
3. Jobs to Be Done 또는 Pain-Gain 매핑
4. 우선순위 결정 (빈도 × 심각도)
5. 디자인 기회 정의

## 안티패턴

- "우리 팀원들에게 물어봤더니..." → 내부자 편향
- 리서치 결과를 이미 정해진 결론을 지지하는 데 사용
- 아웃라이어 의견을 핵심 인사이트로 과장
- 리서치 결과를 공유하지 않고 디자이너만 보유
- 출시 후 리서치 계획 없음

## 실전 팁

- 인터뷰 중 침묵을 두려워하지 말 것. 5초 기다리면 더 깊은 답이 나옴
- 화면 녹화 시 "왜 그렇게 하셨나요?"를 즉시 물어보는 think-aloud 활용
- 리서치 인사이트는 팀 전체가 볼 수 있는 공간에 문서화
- PM/개발자를 관찰 세션에 동행 → 공감대 형성 효과
- Jobs to Be Done 프레임: "~할 때, 나는 ~하고 싶다. 그래서 ~을 할 수 있도록"

## 심화: 리서치 실행 가이드

### 리서치 계획서 필수 항목
1. **리서치 목표**: 무엇을 알고 싶은가
2. **리서치 질문**: 목표를 구체화한 3~5개 질문
3. **방법론**: 인터뷰/설문/관찰 중 선택 이유
4. **참여자 기준**: 누구를 모집할 것인가
5. **일정/예산**: 언제까지, 얼마로
6. **결과물**: 무엇을 만들 것인가 (보고서, 인사이트 문서 등)

### 리서치 참여자 모집 기준
- **스크리너 설문**: 적합한 참여자를 거르는 사전 질문지
- **극단적 사용자 포함**: 헤비 유저 + 비사용자 양쪽 모두
- **다양성 고려**: 연령, 디지털 리터러시, 사용 환경 다양화
- **이해 충돌 배제**: 경쟁사 직원, 지인 제외

### 인터뷰 가이드 작성 원칙
- 오프닝: 목적 설명, 긴장 완화 (5분)
- 배경 질문: 현재 상황, 습관 파악 (10분)
- 핵심 질문: 행동, 동기, 불편함 탐색 (25분)
- 심화 질문: "왜요?", "더 자세히 말씀해주세요" 준비
- 클로징: 놓친 것 확인, 감사 (5분)

### 인사이트 → 디자인 기회 전환
```
관찰: "사용자가 영수증을 촬영해서 엑셀에 수동으로 입력하고 있었다"
인사이트: "현재 지출 기록 방법이 너무 번거로워 포기하는 경우가 많다"
디자인 기회: "영수증 촬영으로 자동 데이터 입력하는 기능"
```

### 리서치 민주화
- 팀원 누구나 기본 리서치를 할 수 있도록 교육
- 표준 인터뷰 템플릿과 가이드 공유
- 인사이트 저장소 (Notion, Dovetail) 팀 전체 접근 가능
- 월 1회 리서치 인사이트 공유 세션 운영

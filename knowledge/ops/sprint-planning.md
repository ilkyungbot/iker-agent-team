# Sprint Planning — IronAct Ops Knowledge

## 개요
2주 단위 스프린트 기반 반복 개발/운영 사이클 관리.

## 스프린트 기본 구조
- 기간: 2주 (월요일 시작 ~ 금요일 종료)
- 스프린트 용량(Capacity): 팀원 × 근무일 × 가용률(80%)
- 스프린트 목표(Sprint Goal): 1~2문장으로 명확히 표현

## 스프린트 이벤트

### Sprint Planning (월요일 오전, 2시간)
**Agenda**
1. 이전 스프린트 미완료 항목 검토 (15분)
2. 백로그 정제 및 우선순위 확인 (30분)
3. 스프린트 목표 설정 (15분)
4. 작업 선택 및 예상 공수 산정 (45분)
5. 담당자 배정 및 확인 (15분)

**산출물**
- Sprint Backlog (확정된 작업 목록)
- Sprint Goal 문서화
- 담당자-작업 매핑 완료

### Daily Standup (매일 오전 9:30, 15분)
**3가지 질문**
1. 어제 무엇을 했나?
2. 오늘 무엇을 할 것인가?
3. 블로커가 있는가?

**규칙**
- 15분 초과 금지 → 심층 논의는 별도 시간
- 블로커 즉시 공유
- 결석 시 Slack으로 텍스트 업데이트

### Sprint Review (격주 금요일 오후, 1시간)
- 완료된 작업 데모
- 이해관계자 피드백 수집
- 미완료 항목 백로그 이동

### Sprint Retrospective (격주 금요일, 45분)
참조: retrospective.md

## 백로그 관리

### 우선순위 기준
```
P0: 즉시 처리 (당일)
P1: 이번 스프린트 내 완료
P2: 다음 스프린트 고려
P3: 미래 백로그
```

### 이슈 작성 기준
```
제목: [컴포넌트] 동작 설명
설명:
  - 배경/컨텍스트
  - 수용 기준 (Acceptance Criteria)
  - 예상 공수
  - 담당자
```

## 스프린트 지표

### 속도(Velocity)
- 완료된 스토리 포인트 / 스프린트
- 3스프린트 평균으로 안정화
- 급격한 변동 시 원인 분석

### 완료율(Completion Rate)
```
완료율 = 완료 항목 / 계획 항목 × 100
목표: 85% 이상
```

### 블로커 해소 시간
- P0 블로커: 4시간 내
- P1 블로커: 24시간 내

## 용량 계산 예시
```
팀원 3명 × 10 근무일 × 6시간(가용) = 180시간
여유분(20%) 제외 → 실제 용량: 144시간
```

## 체크리스트

- [ ] 백로그 정제 완료
- [ ] 이전 스프린트 미완료 항목 처리
- [ ] 스프린트 목표 초안 준비
- [ ] 참석자 일정 확인

### 스프린트 종료 후
- [ ] 완료율 기록
- [ ] 미완료 항목 백로그 이동
- [ ] 속도 업데이트
- [ ] 회고 결과 문서화

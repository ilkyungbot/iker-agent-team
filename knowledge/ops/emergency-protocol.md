# Emergency Protocol — IronAct Ops Knowledge

## 개요
예상치 못한 위기 상황 발생 시 신속하고 체계적으로 대응하는 프로토콜.

## 위기 등급 정의

### Level 1 — Critical (즉시 대응)
**조건**: 서비스 전면 중단 / 주요 클라이언트 계약 해지 위기 / 데이터 유실
**목표 대응 시간**: 30분 내 초기 대응
**에스컬레이션**: 즉시 경영진 보고

### Level 2 — High (4시간 내 대응)
**조건**: SLA 심각한 위반 / 납품 대량 지연 / 클라이언트 강한 불만
**목표 대응 시간**: 4시간 내 대응 계획 수립
**에스컬레이션**: 팀장 보고 후 경영진 판단

### Level 3 — Medium (24시간 내 대응)
**조건**: 단일 납품 지연 / 품질 이슈 / 내부 프로세스 장애
**목표 대응 시간**: 24시간 내 해결 계획
**에스컬레이션**: 팀장 보고

## 위기 대응 프로세스 (REACT)

```
R - Recognize (인지): 상황 감지 및 등급 판단
E - Escalate (에스컬레이션): 적절한 담당자 즉시 알림
A - Assess (평가): 영향 범위 및 원인 분석
C - Communicate (소통): 이해관계자 투명한 상황 공유
T - Take Action (조치): 즉각 대응 + 근본 해결
```

## Level 1 대응 절차

### 0~30분: 초기 대응
1. **인지 즉시**: Ops Lead → 경영진 직통 연락 (전화)
2. **상황 파악**: 영향 범위, 원인 초기 가설
3. **임시 조치**: 피해 확산 차단
4. **클라이언트 통보**: 상황 인지 및 조치 중 안내

### 30분~2시간: 집중 대응
1. **전담 TF 구성** (필요 시)
2. **원인 분석** 병렬 진행
3. **30분 단위 상황 업데이트** (내부)
4. **클라이언트 1시간 간격 업데이트**

### 2시간~해결: 지속 대응
1. **해결책 실행**
2. **영향받은 당사자 사후 케어**
3. **해결 확인 후 공식 종료 선언**

## Level 2 대응 절차

### 0~4시간
1. 팀장 즉시 보고
2. 영향 범위 파악
3. 클라이언트 연락 (Level에 따라)
4. 대응 계획 수립

### 4~24시간
1. 대응 계획 실행
2. 4시간 단위 상황 공유
3. 해결 후 사후 보고

## 클라이언트 긴급 소통 템플릿

### 초기 인지 메시지
```
안녕하세요, [클라이언트명] 담당자님.

현재 [상황 요약]이 발생하여 빠르게 파악하고 있습니다.
[XX시간] 내로 상세 상황과 조치 계획을 공유드리겠습니다.
불편을 드려 죄송합니다.

담당: [이름] / [연락처]
```

### 진행 상황 업데이트
```
[시간] 업데이트

현황: [현재 상태]
원인: [파악된 내용]
조치: [진행 중인 조치]
예상 해결 시간: [시간]

다음 업데이트: [시간]
```

### 해결 완료 메시지
```
[클라이언트명] 담당자님,

[상황]이 [시간]에 해결되었음을 알립니다.

원인: [설명]
조치 내용: [설명]
재발 방지책: [설명]

불편을 드린 점 진심으로 사과드립니다.
추가 확인이 필요하시면 언제든 연락해 주세요.
```

## 사후 처리 (Post-Mortem)

### 48시간 내 실시
1. 타임라인 재구성
2. 근본 원인 분석 (5 Whys)
3. 초기 대응 평가
4. 재발 방지책 수립
5. 문서화 및 공유

### Post-Mortem 문화
- 책임 추궁이 아닌 시스템 개선이 목적
- 익명 참여 허용
- 결과를 팀 학습 자료로 활용

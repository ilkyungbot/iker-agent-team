# Process Improvement — IronAct Ops Knowledge

## 개요
운영 프로세스를 지속적으로 개선하여 효율성과 품질을 높이는 방법론.

## 개선 철학

> "좋은 시스템은 좋은 사람보다 낫다. 하지만 좋은 시스템을 만드는 것은 좋은 사람이다."

## 개선 프레임워크: PDCA

```
Plan (계획) → Do (실행) → Check (확인) → Act (조치)
     ↑_____________________________________|
```

### Plan
- 문제/기회 정의
- 현 상태(As-Is) 분석
- 목표 상태(To-Be) 설정
- 개선 방안 도출

### Do
- 파일럿 실행 (소규모 테스트)
- 결과 측정
- 이슈 기록

### Check
- 계획 대비 결과 비교
- 가설 검증
- 부작용 확인

### Act
- 성공 시: 전체 확대 적용
- 실패 시: 원인 분석 후 Plan 재시작
- 표준화 및 문서화

## 문제 발굴 방법

### 1. 데이터 기반
- KPI 이상 신호 추적
- 반복적 실수/결함 패턴
- SLA 위반 기록
- 재작업 발생 원인

### 2. 팀 피드백
- 회고(Retrospective)에서 나온 불편사항
- 팀원 1:1 미팅
- 익명 설문

### 3. 클라이언트 피드백
- CSAT/NPS 낮은 항목
- 불만 접수 내용
- 반복 요청 패턴

## 근본 원인 분석 (Root Cause Analysis)

### 5 Whys 기법
```
문제: 납품 기한을 3번 연속 지연

Why 1: 왜 지연됐나? → 검수 시간 부족
Why 2: 왜 검수 시간이 부족했나? → 검수 단계가 납품 직전에 몰림
Why 3: 왜 직전에 몰렸나? → 제작 완료 후 바로 검수 착수
Why 4: 왜 버퍼가 없었나? → 일정에 검수 시간이 미포함
Why 5: 왜 미포함됐나? → 일정 템플릿에 검수 단계 없음

근본 원인: 일정 템플릿에 검수 단계 누락
해결책: 모든 프로젝트 일정에 검수 버퍼 2일 추가
```

### 피쉬본 다이어그램 (복잡한 문제)
카테고리: 사람 / 프로세스 / 도구 / 환경 / 측정 / 재료

## 개선 우선순위 결정

### Impact vs Effort 매트릭스
```
         낮은 Effort    높은 Effort
높은 Impact | Quick Win  | Major Project
낮은 Impact | Fill-In    | Thankless Task
```

→ Quick Win 먼저, Major Project 계획적으로

### 개선 항목 평가 기준
- 영향받는 팀원/클라이언트 수
- 현재 비효율로 인한 시간/비용 손실
- 구현 복잡도
- 유지보수 비용

## 개선 실행 프로세스

1. **아이디어 수집** (상시)
   - Notion 개선 아이디어 백로그에 누구든 등록

2. **월간 검토** (매월 첫째 주)
   - 백로그 검토 및 우선순위 정렬
   - 이번 달 진행할 1~3개 선정

3. **파일럿** (2~4주)
   - 소규모 적용
   - 측정 지표 사전 정의

4. **평가 및 결정** (파일럿 종료 후 1주)
   - 확대 / 수정 후 재시도 / 중단

5. **표준화** (채택 시)
   - 프로세스 문서 업데이트
   - 팀 공유 및 교육

## 개선 추적

### 개선 효과 측정
```
개선 전 → 개선 후 비교

- 오류율 변화
- 비용 절감액
- 팀 만족도 변화
```

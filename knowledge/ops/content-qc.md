# Content QC — IronAct Ops Knowledge

## 개요
콘텐츠 품질을 일관되게 보장하기 위한 검수 프로세스와 기준.

## QC 프로세스 개요

```
제작 → 1차 자체 검토 → QC 검수 → 클라이언트 검토 → 최종 납품
```

### 단계별 책임
| 단계 | 담당 | SLA |
|------|------|-----|
| 1차 자체 검토 | 제작자 | 납품 전 필수 |
| QC 검수 | QC 담당자 | 4~8시간 |
| 클라이언트 검토 | 클라이언트 AM | 합의된 기한 |
| 최종 승인 | PM | 납품 당일 |

## 콘텐츠 유형별 QC 기준

### 카피/텍스트 콘텐츠
**필수 확인 항목**
- [ ] 맞춤법 및 문법 (한글 맞춤법 검사기 활용)
- [ ] 브랜드 보이스 및 톤앤매너 준수
- [ ] 클라이언트 금지어/필수어 반영
- [ ] 사실 확인 (가격, 날짜, 수치 등)
- [ ] CTA(행동 유도) 문구 명확성
- [ ] 법적 주의사항 포함 여부

### 이미지/디자인 콘텐츠
**필수 확인 항목**
- [ ] 해상도 및 파일 형식 (스펙 준수)
- [ ] 브랜드 컬러 팔레트 준수
- [ ] 폰트 가이드라인 준수
- [ ] 여백 및 레이아웃 가이드 준수
- [ ] 이미지 저작권 확인
- [ ] 텍스트 가독성 (대비율)
- [ ] 클라이언트 로고 최신 버전 사용

### 영상 콘텐츠
**필수 확인 항목**
- [ ] 해상도 및 포맷 (플랫폼 스펙)
- [ ] 자막 정확성 및 타이밍
- [ ] 음향 품질 (배경음, 목소리)
- [ ] 브랜딩 요소 (인트로/아웃트로)
- [ ] 저작권 음악 사용 여부
- [ ] 런닝타임 스펙 준수

### SNS 콘텐츠
**필수 확인 항목**
- [ ] 플랫폼별 이미지 사이즈
- [ ] 해시태그 적절성 및 수량
- [ ] 링크 작동 확인
- [ ] 발행 일시 정확성
- [ ] 멘션 계정 존재 여부

## QC 피드백 작성 기준

### 피드백 형식
```
[위치]: 콘텐츠 내 위치 (예: 3번째 문단, 하단 이미지)
[유형]: 오류 / 개선 / 확인 필요
[현재]: 현재 내용
[수정안]: 제안 수정 내용 또는 방향
[우선순위]: 필수 / 권장
```

### 우선순위 기준
- **필수(Must Fix)**: 오탈자, 사실 오류, 법적 문제, 브랜드 가이드 위반
- **권장(Should Fix)**: 가독성, 톤, 레이아웃 최적화
- **선택(Nice to Have)**: 미세 조정, 취향 차이

## 승인/반려 기준

### 즉시 반려 사유
- 사실 관계 오류 (가격, 날짜, 수치)
- 브랜드 가이드 심각한 위반
- 법적 문제 소지
- 스펙 미달 (해상도, 포맷 등)
- 클라이언트 금지어 사용

### 조건부 승인
- 마이너 수정 사항만 남은 경우
- 수정 후 재검토 없이 납품 가능

## QC 기록 관리
- 모든 QC 피드백 Notion에 기록
- 반복 오류 패턴 월간 분석
- 품질 트렌드 보고서 반영

## 체크리스트

### QC 시작 전
- [ ] 브리프 및 가이드라인 재확인
- [ ] 이전 피드백 이력 확인
- [ ] QC 체크리스트 템플릿 준비

### QC 완료 후
- [ ] 피드백 Notion 기록
- [ ] 제작팀에 피드백 공유
- [ ] 긴급 수정 건 별도 표기

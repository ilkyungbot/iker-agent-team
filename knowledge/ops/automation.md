# Automation — IronAct Ops Knowledge

## 개요
반복 작업을 자동화하여 팀의 고부가가치 업무 집중도를 높이는 전략.

## 자동화 대상 선정 기준

### 자동화 적합 조건 (3R)
- **Repetitive**: 동일한 작업이 반복됨 (주 3회 이상)
- **Rule-based**: 명확한 규칙/로직이 있음
- **Reliable**: 오류 허용 범위가 낮음

### 자동화 우선순위
```
높음: 시간 소모 크고 + 오류 가능성 높은 반복 작업
중간: 시간 소모 크고 + 오류 가능성 낮은 반복 작업
낮음: 시간 소모 작은 반복 작업
제외: 판단/창의가 필요한 작업
```

## 현재 자동화 가능 영역

### 커뮤니케이션 자동화
- **정기 보고 알림**: Slack 봇 → 주간 보고 리마인더
- **온보딩 체크리스트**: 신규 클라이언트 등록 시 자동 생성
- **만료 알림**: 계약 만료 D-90/60/30 자동 알림
- **생일/기념일**: 클라이언트 주요 날짜 알림

### 데이터 수집 자동화
- **KPI 집계**: Linear API → 스프린트 완료율 자동 계산
- **SLA 모니터링**: 응답 시간 자동 추적
- **보고서 초안**: 템플릿 + 데이터 자동 조합

### 워크플로우 자동화
- **이슈 라우팅**: 태그/카테고리별 자동 담당자 배정
- **상태 업데이트**: 이슈 완료 시 클라이언트 알림 자동 발송
- **파일 정리**: 납품 완료 시 폴더 자동 이동

## 자동화 도구 스택

### 현재 활용 가능
| 도구 | 용도 | 난이도 |
|------|------|--------|
| Slack Workflow | 알림/폼 자동화 | 낮음 |
| Notion Formula | 데이터 계산 | 낮음 |
| Google Sheets 수식 | KPI 집계 | 낮음 |
| Zapier/Make | 도구 간 연동 | 중간 |
| Python 스크립트 | 복잡한 데이터 처리 | 높음 |
| Claude (AI) | 문서 생성/분석 | 낮음 |

## 자동화 구현 프로세스

### Step 1: 문서화 먼저
현재 수동 프로세스를 단계별로 기록
→ "이것을 자동화한다면 어떻게 될까?" 질문

### Step 2: 소규모 테스트
- 1~2주 파일럿
- 수동 대안 병행 유지
- 예상치 못한 케이스 기록

### Step 3: 안전장치 설계
- 자동화 실패 시 폴백(Fallback) 프로세스
- 모니터링 알림 설정
- 수동 개입 포인트 명시

### Step 4: 문서화 및 교육
- 자동화 동작 방식 문서화
- 이슈 발생 시 대응 가이드
- 담당자 지정

## 자동화 관리

### 정기 점검 (월 1회)
- 자동화 정상 작동 여부 확인
- 오류/예외 케이스 검토
- 최적화 기회 탐색

### 자동화 인벤토리

|----------|------|--------|-------------|
| 주간 보고 알림 | Slack | - | - |
| 계약 만료 알림 | - | - | - |

## 주의사항
- 자동화는 사람을 대체하는 것이 아니라 보조하는 것
- 중요한 의사결정은 반드시 사람이
- 자동화 실패를 전제로 백업 계획 수립
- 과도한 자동화는 오히려 복잡도 증가

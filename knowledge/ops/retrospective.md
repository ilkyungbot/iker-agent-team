# Retrospective — IronAct Ops Knowledge

## 개요
팀과 프로젝트를 지속적으로 개선하는 정기 회고 프레임워크.

## 회고의 목적
- 잘된 것을 인식하고 강화
- 개선 기회를 발굴하고 실행
- 팀 신뢰와 심리적 안전감 구축
- 반복 실수 방지

## 회고 유형

### 1. 스프린트 회고 (격주)
**참석**: 팀 전원
**시간**: 45~60분
**목적**: 작업 방식 개선

### 2. 프로젝트 회고 (프로젝트 종료 시)
**참석**: 프로젝트 관련자
**시간**: 60~90분
**목적**: 프로젝트 레슨런 추출

### 3. 분기 회고 (분기별)
**참석**: 팀 + 리더십
**시간**: 2~3시간
**목적**: 분기 성과 평가 + 다음 분기 방향

## 스프린트 회고 진행 방법

### 준비 (미팅 전)
- 스프린트 데이터 준비 (완료율, KPI 현황)
- 회고 보드 세팅 (Notion 또는 FigJam)
- 심리적 안전감 조성 공지

### 진행 (45~60분)

#### Format 1: Start / Stop / Continue
```
Start: 새로 시작해야 할 것 (5분 작성 + 10분 공유)
Stop: 멈춰야 할 것 (5분 작성 + 10분 공유)
Continue: 계속해야 할 것 (5분 작성 + 5분 공유)
→ 액션 아이템 결정 (10분)
```

#### Format 2: 4Ls
```
Liked: 좋았던 것
Learned: 배운 것
Lacked: 부족했던 것
Longed For: 바랐던 것
```

#### Format 3: 감사 → 개선
```
감사한 점 (개인/팀원/프로세스): 3가지씩
개선할 점: 팀 전체에서 2~3개 선정
```

### 마무리
- 액션 아이템 확정 (담당자 + 기한)
- Linear에 등록
- 다음 회고 전 진행 상황 확인 약속

## 프로젝트 회고 (Post-Project)

### 타임라인 재구성
```
시작 → 주요 사건 → 이슈 포인트 → 종료
```

### 핵심 질문
1. 무엇이 계획대로 됐는가?
2. 무엇이 예상과 달랐는가?
3. 그 이유는?
4. 다음 유사 프로젝트에 어떻게 반영할 것인가?

### 산출물
- 레슨런(Lessons Learned) 문서
- 프로세스 업데이트 사항
- 재사용 가능한 템플릿/산출물

## 회고 진행 팁

### 심리적 안전감 조성
- 비난 없는 문화 강조 (Non-blame)
- 이름 없이 이슈 논의 가능하도록
- 모든 의견 동등하게 취급

### 참여도 높이기
- 사전 익명 설문 (선호 시)
- 모든 팀원 발언 기회 보장
- 에너자이저(짧은 워밍업 활동)

### 실행 가능성 확보
- 액션 아이템 최대 3개 (집중)
- 다음 스프린트 내 실행 가능한 것
- 측정 가능한 결과 정의

## 회고 기록 및 추적

### 회고 문서 구조
```
# [기간] 회고 — YYYY-MM-DD

참석자:
진행 형식:

## 잘된 점
-

## 개선 필요
-

## 액션 아이템
| 항목 | 담당자 | 기한 | 상태 |
|------|--------|------|------|

## 이전 회고 액션 확인
| 항목 | 담당자 | 완료 여부 |
|------|--------|-----------|
```

### 패턴 분석 (월간)
- 반복 등장하는 개선 사항 → 구조적 문제
- 한 번 등장 후 해소된 사항 → 정상
- 개선됐다가 재발한 사항 → 근본 해결 필요

## 회고 효과 측정
```
회고 품질 지표:
- 액션 아이템 실행률 (목표: 80%+)
- 동일 이슈 재발률 (목표: 감소 트렌드)
- 팀 만족도 (3개월 전 대비)
```

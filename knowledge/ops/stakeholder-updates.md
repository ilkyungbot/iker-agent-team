# Stakeholder Updates — IronAct Ops Knowledge

## 개요
이해관계자(경영진, 클라이언트, 팀 파트너)에게 적시에 정확한 정보를 전달하는 체계.

## 이해관계자 맵핑

### 내부 이해관계자
| 그룹 | 관심사 | 업데이트 주기 | 채널 |
|------|--------|---------------|------|
| 경영진 | 비즈니스 임팩트, 리스크 | 주간 | 보고서 + 미팅 |
| 타 팀장 | 협업 의존성 | 필요 시 | Slack |
| 팀원 | 목표, 결정 이유 | 주간 | 팀 미팅 + Slack |

### 외부 이해관계자
| 그룹 | 관심사 | 업데이트 주기 | 채널 |
|------|--------|---------------|------|
| Tier 1 클라이언트 | 성과, 진행 현황 | 주간 | 미팅 + 이메일 |
| Tier 2 클라이언트 | 납품 현황 | 격주 | 이메일 |
| Tier 3 클라이언트 | 결과 | 월간 | 이메일 |
| 벤더/파트너 | 요청사항, 일정 | 필요 시 | 이메일/Slack |

## 업데이트 원칙

### BLUF (Bottom Line Up Front)
가장 중요한 메시지를 먼저, 세부 사항은 후에.

### 5가지 원칙
1. **적시성**: 필요할 때 필요한 정보를
2. **정확성**: 검증된 정보만 공유
3. **명확성**: 모호함 없이 구체적으로
4. **간결성**: 핵심만, 불필요한 내용 제거
5. **행동 유도성**: 다음 액션 항상 명시

## 이해관계자별 업데이트 형식

### 경영진 주간 보고
```
제목: [주간 운영 보고] YYYY-MM-DD 주차

📊 이번 주 핵심 지표
- 납품 완료: X건 (목표 대비 XX%)
- SLA 준수율: XX%
- 클라이언트 이슈: X건

🎯 주요 성과
- [항목 1]
- [항목 2]

⚠️ 이슈 및 리스크
- [이슈 1]: 상태 / 조치 계획
- [이슈 2]: 상태 / 조치 계획

📅 다음 주 계획
- [주요 마일스톤]
- [주의가 필요한 사항]
```

### 클라이언트 주간 업데이트 (Tier 1)
```
안녕하세요, [이름]님.

이번 주 진행 현황을 공유드립니다.

✅ 완료된 항목
- [항목 1]

🔄 진행 중
- [항목 1] (예상 완료: [날짜])

📋 다음 주 계획
- [항목 1]

❓ 확인이 필요한 사항
- [항목 1] → 회신 기한: [날짜]

감사합니다.
```

- [항목]

- [블로커/리스크]

## 나쁜 소식 전달 (Bad News Update)

### 원칙
1. **즉시 공유**: 나쁜 소식일수록 빠르게
2. **사실만**: 추측이나 변명 없이
3. **책임 인정**: 필요한 경우
4. **다음 스텝**: 솔루션 또는 계획 함께 제시

### 템플릿
```
[솔직한 제목]: [상황 요약]

현황:
[무슨 일이 발생했는지 사실 중심]

영향:
[이로 인한 영향]

원인:
[현재까지 파악된 원인]

조치 중인 사항:
[현재 취하고 있는 액션]

다음 업데이트:
[언제, 어떤 내용으로]
```

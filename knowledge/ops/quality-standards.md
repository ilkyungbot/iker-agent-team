# Quality Standards — IronAct Ops Knowledge

## 개요
IronAct 운영 전반에 적용되는 품질 기준과 표준 정의.

## 품질 철학

> "품질은 결과물에 있는 것이 아니라 프로세스에 있다."

품질은 사후 검수가 아니라 제작 단계부터 내재화된다.

## 품질 4대 기준

### 1. 정확성 (Accuracy)
- 사실 관계 오류 제로
- 수치, 날짜, 이름 이중 확인
- 출처 검증 가능한 정보만 사용

### 2. 일관성 (Consistency)
- 브랜드 가이드라인 100% 준수
- 동일 프로젝트 내 톤앤매너 통일
- 반복 요소(로고, 색상, 폰트) 표준화

### 3. 적시성 (Timeliness)
- 약속된 납품 기한 준수율 ≥ 95%
- 피드백 반영 속도 ≥ 24시간 내
- 긴급 요청 대응 ≤ 4시간

### 4. 완결성 (Completeness)
- 브리프 요구사항 100% 반영 확인
- 누락 항목 체크리스트 활용
- 납품 전 완결성 자가 점검

## 산출물별 품질 기준

### 문서/보고서
| 기준 | 방법 | 목표 |
|------|------|------|
| 맞춤법 | 자동 검사기 + 육안 | 오류 0 |
| 데이터 정확성 | 이중 확인 | 오류 0 |
| 완결성 | 체크리스트 | 100% |
| 가독성 | 동료 리뷰 | 3분 내 핵심 파악 |

### 디자인 산출물
| 기준 | 방법 | 목표 |
|------|------|------|
| 브랜드 준수 | 가이드라인 대조 | 100% |
| 기술 스펙 | 자동/수동 확인 | 100% |
| 저작권 | 소스 확인 | 위반 0 |

### 커뮤니케이션 산출물
| 기준 | 방법 | 목표 |
|------|------|------|
| 응답 SLA | 모니터링 | ≥ 98% |
| 정확성 | 팩트체크 | 오류 0 |
| 완결성 | 요청사항 대조 | 100% |

## 품질 측정 지표

### 결함율 (Defect Rate)
```
결함율 = 결함 건 / 전체 납품 건 × 100
목표: ≤ 5%
```

### 재작업율 (Rework Rate)
```
재작업율 = 수정 요청 건 / 전체 납품 건 × 100
목표: ≤ 15%
```

### 1차 승인율 (First Pass Yield)
```
FPY = 수정 없이 승인된 건 / 전체 납품 건 × 100
목표: ≥ 80%
```

## 품질 개선 사이클

```
측정 → 분석 → 개선 → 표준화 → 측정...
```

### 월간 품질 리뷰
1. 결함율, 재작업율 집계
2. 반복 오류 패턴 분석
3. 근본 원인 파악 (5 Whys)
4. 개선 액션 수립
5. 프로세스/체크리스트 업데이트

## 품질 문화 구축

### 팀원 역할
- **모든 팀원**: 자체 QC 후 제출 (내부 고객 마인드)
- **QC 담당자**: 공정하고 구체적인 피드백
- **PM**: 품질 기준 명확화 및 최종 책임

### 피드백 문화
- 결과물이 아닌 프로세스에 집중
- 구체적 개선안 제시 (비판만 하지 않기)
- 잘된 점도 인정

## 표준 템플릿 관리
- 위치: Notion > 템플릿 라이브러리
- 업데이트 주기: 분기별
- 책임자: Ops Lead

# Reporting — IronAct Ops Knowledge

## 개요
데이터를 의미 있는 인사이트로 변환하여 의사결정을 지원하는 보고 체계.

## 보고서 유형

### 1. 주간 운영 보고 (Weekly Ops Report)
**대상**: 팀 리더, 내부 이해관계자
**형식**: Notion 페이지 or Slack 요약
**분량**: 1~2페이지

**구성**
```
1. 이번 주 주요 성과 (3줄 이내)
2. KPI 현황 (목표 vs 실적 테이블)
3. 진행 중 주요 프로젝트 상태
4. 이슈 및 리스크
5. 다음 주 계획
```

### 2. 월간 성과 보고 (Monthly Performance Report)
**대상**: 경영진, 클라이언트
**형식**: Notion 슬라이드 or PDF
**분량**: 5~10페이지

**구성**
```
1. Executive Summary (1페이지)
2. KPI 달성 현황 (시각화)
3. 주요 프로젝트 진행 현황
4. 클라이언트 만족도
5. 이슈 및 해결 현황
6. 다음 달 목표
```

### 3. 분기 비즈니스 리뷰 (QBR)
**대상**: 경영진, 주요 클라이언트
**형식**: 발표 자료 (30~45분)
**분량**: 15~20슬라이드

**구성**
```
1. 분기 하이라이트
2. OKR 달성도
3. 비즈니스 임팩트
4. 개선된 영역
5. 챌린지 및 대응
6. 다음 분기 전략
```

### 4. 클라이언트 보고
**대상**: 개별 클라이언트
**형식**: 클라이언트별 맞춤
**주기**: 주간 또는 월간

**구성**
```
1. 이번 기간 완료 사항
2. 성과 지표 (약속된 KPI)
3. 진행 중 항목
4. 다음 기간 계획
5. 요청/논의 사항
```

## 보고서 작성 원칙

### 피라미드 원칙 (Minto Pyramid)
```
결론 (가장 중요한 메시지)
  ├─ 근거 1
  ├─ 근거 2
  └─ 근거 3
```
→ 결론을 먼저, 세부 내용은 뒤에

### 데이터 시각화 원칙
- 표: 정확한 수치 비교 시
- 선 그래프: 시간에 따른 트렌드
- 막대 그래프: 카테고리 비교
- 색상: 신호등 체계 (🟢🟡🔴)

### 글쓰기 원칙
1. 사실과 해석 분리
2. 수치 항상 맥락과 함께 (전월 대비 +5%)
3. 액션 아이템 명시
4. 수신자 눈높이에 맞는 언어

## 보고 자동화

### 자동 수집 대상
- Linear 이슈 완료율
- 스프린트 번다운
- SLA 준수율

### 자동화 도구
- Notion 데이터베이스 자동 집계
- Slack 주간 요약 봇 (가능한 경우)
- 스프레드시트 수식 활용

## 보고 캘린더

|--------|-----------|-----------|

# Project Management — IronAct Ops Knowledge

## 개요
프로젝트 생애주기 전반을 체계적으로 관리하는 원칙과 프로세스.

## 프로젝트 생애주기
```
Initiation → Planning → Execution → Monitoring → Closure
```

### 1. 착수 (Initiation)
- 프로젝트 헌장(Project Charter) 작성
  - 목적 및 목표 명확화
  - 핵심 이해관계자 목록
  - 예산 및 일정 대략적 범위
  - 성공 기준(Success Criteria) 정의
- 킥오프 미팅 준비 및 진행

### 2. 계획 (Planning)
- WBS(Work Breakdown Structure) 작성
- 마일스톤 정의 (M1, M2, M3...)
- 리소스 배정 (담당자, 예상 공수)
- 리스크 사전 식별 및 대응 계획
- 커뮤니케이션 계획 수립

### 3. 실행 (Execution)
- 일일 스탠드업 또는 주간 동기화
- 작업 진행 상황 실시간 추적
- 블로커(Blocker) 즉시 에스컬레이션
- 변경 요청(Change Request) 관리

### 4. 모니터링 (Monitoring)
- KPI 대시보드 주간 업데이트
- 일정 편차(Schedule Variance) 분석
- 예산 소진율(Burn Rate) 추적
- 품질 체크포인트 정기 시행

### 5. 종료 (Closure)
- 산출물 최종 인수인계
- 회고(Retrospective) 진행
- 레슨런(Lessons Learned) 문서화
- 프로젝트 공식 종료 선언

## 우선순위 프레임워크

### MoSCoW
- **Must Have**: 없으면 프로젝트 실패
- **Should Have**: 중요하지만 없어도 런칭 가능
- **Could Have**: 있으면 좋음
- **Won't Have**: 이번 범위 제외

### RICE 스코어
```
RICE = (Reach × Impact × Confidence) / Effort
```

## 일반 원칙
1. 모든 작업은 담당자 1명, 마감일 1개
2. 블로커는 24시간 내 에스컬레이션
3. 범위 변경은 반드시 문서화
4. 주간 상태 보고는 금요일 EOD
5. 위험 신호(Red Flag)는 즉시 공유

# SLA Management — IronAct Ops Knowledge

## 개요
서비스 수준 협약(Service Level Agreement)을 정의, 관리, 준수하는 체계.

## SLA 정의

### SLA란?
서비스 제공자와 클라이언트 간 서비스 수준에 관한 공식 합의.
- 무엇을 (What): 서비스 범위
- 얼마나 잘 (How Well): 품질 기준
- 얼마나 빨리 (How Fast): 응답/납품 시간
- 못 지키면 (Consequence): 위반 시 처리

## 표준 SLA 항목

### 응답 시간 SLA
| 요청 유형 | 응답 기한 | 해결 기한 |
|-----------|-----------|-----------|
| 긴급 문의 (장애) | 1시간 | 4시간 |
| 일반 문의 | 4시간 (업무일) | 24시간 |
| 비긴급 요청 | 24시간 | 3영업일 |

### 납품 SLA
| 작업 유형 | 기준 납품 기한 | 버퍼 |
|-----------|---------------|------|
| 표준 콘텐츠 | 브리핑 후 5영업일 | 없음 |
| 긴급 콘텐츠 | 브리핑 후 2영업일 | 없음 |
| 대형 프로젝트 | 별도 협의 | 10% |

### 품질 SLA
| 지표 | 목표값 |
|------|--------|
| 1차 승인율 | ≥ 80% |
| 수정 후 승인율 | ≥ 95% |
| 최대 수정 횟수 | 3회 이하 |

### 가용성 SLA
- 업무 가능 시간: 월~금 09:00~18:00 (한국시간)
- 긴급 연락 가능 시간: 07:00~22:00
- 공휴일: 대한민국 법정 공휴일 적용

## SLA 측정 및 추적

### 측정 방법
```
응답시간 SLA 준수율 = 기한 내 응답 건 / 전체 요청 건 × 100
납품 SLA 준수율 = 기한 내 납품 건 / 전체 납품 건 × 100
```

### 추적 주기
- 일간: SLA 위반 임박 건 모니터링
- 주간: 주간 준수율 집계
- 월간: 월간 SLA 성과 보고서

### 위반 임박 알림
- 응답 SLA: 기한 1시간 전
- 납품 SLA: 기한 24시간 전

## SLA 위반 처리

### 위반 발생 시
1. **즉시 인지** → 담당자 알림
2. **클라이언트 사전 통보** (위반 전 가능하다면)
3. **원인 기록** (내부)
4. **클라이언트에 사과 및 조치 계획**
5. **재발 방지책 수립**

### 위반 보고 형식
```
위반 내용: [SLA 항목]
발생 일시: [날짜/시간]
클라이언트: [이름]
원인: [설명]
조치 사항: [설명]
재발 방지: [계획]
```

### 반복 위반 처리
- 동일 사유 3회 이상: 프로세스 개선 의무화
- 동일 클라이언트 2회 이상: 사과 + 보상 방안 검토

## SLA 커스터마이징

### Tier별 기본 SLA
| Tier | 응답 시간 | 납품 기한 | 추가 특전 |
|------|-----------|-----------|-----------|
| Tier 1 | 1시간 | 표준 -1일 | 전담 AM |
| Tier 2 | 4시간 | 표준 | AM 공유 |
| Tier 3 | 24시간 | 표준 | 없음 |

### 특별 SLA 합의 시
- 반드시 문서화
- 운영팀 전체 공유
- Linear 이슈에 SLA 태그 추가

## SLA 문서 관리
- 위치: Notion > 클라이언트 > [클라이언트명] > 계약/SLA
- 서명 사본 보관 필수
- 갱신 시 버전 관리 (v1.0, v1.1...)

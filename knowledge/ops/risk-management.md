# Risk Management — IronAct Ops Knowledge

## 개요
운영상 리스크를 사전에 식별, 평가, 대응하여 사업 연속성을 보장.

## 리스크 관리 사이클
```
식별 → 평가 → 대응 계획 → 모니터링 → 검토
```

## 리스크 평가 매트릭스

### 발생 가능성 (Probability)
| 등급 | 정의 |
|------|------|
| 5 (매우 높음) | 6개월 내 발생 가능성 > 70% |
| 4 (높음) | 6개월 내 발생 가능성 40~70% |
| 3 (중간) | 1년 내 발생 가능성 20~40% |
| 2 (낮음) | 발생 가능성 5~20% |
| 1 (매우 낮음) | 발생 가능성 < 5% |

### 영향도 (Impact)
| 등급 | 정의 |
|------|------|
| 5 (치명적) | 사업 운영 중단 |
| 4 (심각) | 주요 클라이언트 이탈 / 매출 20%+ 감소 |
| 3 (보통) | 납품 지연 / 클라이언트 불만 |
| 2 (경미) | 내부 비효율 / 재작업 |
| 1 (미미) | 무시 가능한 수준 |

### 리스크 점수
```
리스크 점수 = 발생 가능성 × 영향도

15~25: 🔴 High Risk — 즉시 대응 계획 수립
8~14:  🟡 Medium Risk — 모니터링 + 대응 계획
1~7:   🟢 Low Risk — 주기적 모니터링
```

## 대응 전략

### 회피 (Avoid)
리스크 자체를 제거하거나 발생 원인 제거
예) 단일 클라이언트 매출 의존도 30% 이하 유지

### 완화 (Mitigate)
발생 가능성 또는 영향도 줄이기
예) 핵심 인력 지식 문서화 → 이탈 시 영향 완화

### 전가 (Transfer)
리스크를 제3자에게 이전
예) 보험 가입, 계약서 배상 조항

### 수용 (Accept)
리스크 감수, 비용 대비 대응 불필요
예) 저점수 리스크는 모니터링만

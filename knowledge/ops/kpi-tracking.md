# KPI Tracking — IronAct Ops Knowledge

## 개요
핵심 성과 지표를 체계적으로 정의, 측정, 관리하는 프레임워크.

## KPI 설계 원칙

### SMART 기준
- **S**pecific: 구체적인 지표
- **M**easurable: 측정 가능한 수치
- **A**chievable: 달성 가능한 목표값
- **R**elevant: 비즈니스 목표와 연관
- **T**ime-bound: 측정 기간 명시

### KPI 계층 구조
```
Company OKR
  └─ Team KPI
      └─ Individual Metric
```

## 운영 핵심 KPI

### 납품 품질
| 지표 | 정의 | 목표 |
|------|------|------|
| 정시 납품율 | 기한 내 납품 / 전체 납품 | ≥ 95% |
| 초안 승인율 | 1차 승인 / 전체 요청 | ≥ 80% |
| 수정 횟수 | 납품 건당 평균 수정 | ≤ 1.5회 |

### 클라이언트 성과
| 지표 | 정의 | 목표 |
|------|------|------|
| CSAT | 클라이언트 만족도 설문 | ≥ 4.2/5.0 |
| NPS | 순 추천 지수 | ≥ 40 |
| Retention Rate | 갱신 클라이언트 / 만료 클라이언트 | ≥ 85% |
| Churn Rate | 이탈 클라이언트 비율 | ≤ 10%/분기 |

### 운영 효율성
| 지표 | 정의 | 목표 |
|------|------|------|
| SLA 준수율 | SLA 충족 건 / 전체 건 | ≥ 98% |
| 블로커 해소 시간 | 블로커 등록~해소 시간 | P0: 4h, P1: 24h |
| 스프린트 완료율 | 완료 항목 / 계획 항목 | ≥ 85% |

### 재무 성과
| 지표 | 정의 | 목표 |
|------|------|------|
| MRR | 월 반복 수익 | 목표치 별도 |
| 계약 갱신율 | - | ≥ 85% |
| Upsell Rate | 업셀 클라이언트 비율 | ≥ 20%/분기 |

## 측정 주기

### 일간 (Daily)
- 블로커 수 및 해소 현황
- SLA 위반 임박 건
- 당일 납품 예정 건

### 주간 (Weekly)
- 스프린트 진행률
- 정시 납품율
- 주요 이슈 현황

### 월간 (Monthly)
- 전체 KPI 대시보드 업데이트
- CSAT/NPS 수집
- 재무 지표 정산
- 트렌드 분석

### 분기 (Quarterly)
- OKR 달성도 평가
- KPI 목표값 재검토
- QBR 자료 준비

## KPI 추적 방법

### 데이터 소스
- Linear: 작업 완료/지연 데이터
- Notion: 클라이언트 만족도 기록
- Slack: 응답 시간 (수동 추적)
- 별도 스프레드시트: 재무 지표

### 대시보드 구성
```
섹션 1: 이번 주 핵심 수치 (3개 이내)
섹션 2: 트렌드 그래프 (4주)
섹션 3: 목표 대비 실적
섹션 4: 이슈 및 액션 아이템
```

## KPI 이상 신호 대응

### 임계값 설정
```
정상: 목표의 95~105%
주의: 목표의 80~94%
위험: 목표의 80% 미만
```

### 위험 신호 발생 시
1. 원인 분석 (2일 내)
2. 개선 계획 수립
3. 이해관계자 공유
4. 주간 추적 강화

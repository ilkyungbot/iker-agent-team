# Resource Allocation — IronAct Ops Knowledge

## 개요
팀 리소스(시간, 사람, 예산)를 최적으로 배분하여 최고의 아웃풋을 이끌어내는 방법.

## 리소스 유형

### 인적 리소스 (Human Resources)
- 팀원별 역량 및 가용 시간
- 전문성 영역 매핑
- 현재 워크로드 가시화

### 시간 리소스 (Time Resources)
- 팀 전체 가용 시간 = 팀원 수 × 근무일 × 가용률(80%)
- 반복 업무 vs 프로젝트 업무 비율
- 버퍼 시간 확보 (전체의 20%)

### 예산 리소스 (Budget Resources)
- 프로젝트별 예산 배분
- 벤더/외주 비용
- 도구 구독 비용

## 리소스 가시화

### 팀 용량 계획 (Capacity Planning)

```
주간 팀 가용 시간:
팀원 A: [XX]시간 (현재 XX% 사용)
팀원 B: [XX]시간 (현재 XX% 사용)
팀원 C: [XX]시간 (현재 XX% 사용)
────────────────────
전체: [XX]시간 / 가용: [XX]시간
```

### 리소스 사용률 기준
| 사용률 | 상태 | 조치 |
|--------|------|------|
| 60~80% | 최적 | 유지 |
| 80~90% | 주의 | 모니터링 강화 |
| 90%+ | 위험 | 우선순위 조정 또는 외주 검토 |
| 60% 미만 | 여유 | 추가 프로젝트 검토 |

## 리소스 배분 의사결정

### 우선순위 기반 배분
```
1. 기존 클라이언트 SLA 이행 (최우선)
2. 신규 클라이언트 온보딩
3. 전략적 프로젝트
4. 내부 개선 활동
5. R&D / 학습
```

### 배분 시 고려사항
- 역량 매칭 (작업 요건 ↔ 팀원 스킬)
- 성장 기회 균형 (도전적 작업 배분)
- 팀원 선호도 (가능한 범위 내)
- 팀원 컨디션 및 번아웃 방지

## 리소스 충돌 해소

### 충돌 발생 시
1. 영향받는 프로젝트/작업 목록화
2. 비즈니스 임팩트 기준 재우선순위
3. 저우선순위 작업 지연/축소 협의
4. 외주/지원 필요 시 에스컬레이션

### 피크 시즌 대응
- 사전 예측 (역사적 패턴 분석)
- 선제적 외주 벤더 확보
- 미리 백로그 정리
- 부가 작업 일시 중단

## 외주 활용 기준

### 외주 고려 시점
- 팀 용량이 2주 이상 90% 초과 예상
- 특수 전문성 일시적으로 필요
- 반복적이지 않은 단발성 작업

### 외주 비용 판단
```
외주 비용 < 내부 처리 시 기회비용 → 외주 합리적
외주 비용 > 내부 처리 시 기회비용 → 내부 처리
```

## 예산 관리

### 월간 예산 추적
- 계획 대비 실제 집행 비교
- 카테고리별 소진율
- 초과 예상 시 즉시 보고

### 예산 배분 원칙
- 클라이언트 프로젝트: 계약 금액의 X% 이내
- 내부 도구: 연간 예산 계획 내
- 긴급 예비비: 전체 예산의 5~10%

# Team Coordination — IronAct Ops Knowledge

## 개요
팀원 간 효과적인 협업과 정렬(Alignment)을 유지하는 운영 방법.

## 팀 구조 및 역할 명확화

### RACI 모델
| 역할 | 정의 |
|------|------|
| **R**esponsible | 실제 작업 수행자 |
| **A**ccountable | 최종 책임자 (1명) |
| **C**onsulted | 의견 제공자 |
| **I**nformed | 결과 공유 대상 |

→ 모든 주요 작업에 RACI 명시. A(책임자)는 반드시 1명.

## 정기 동기화 구조

### 주간 팀 미팅 (월요일, 1시간)
**목적**: 주간 목표 정렬, 블로커 조기 발굴
```
1. 지난 주 결과 공유 (10분)
2. 이번 주 목표 확인 (15분)
3. 리소스/의존성 파악 (15분)
4. 블로커 논의 (15분)
5. 공지/기타 (5분)
```

### 격주 1:1 미팅 (팀원별, 30분)
**목적**: 개인 성장, 업무 장애물 해소, 관계 구축
```
1. 팀원 근황/컨디션 (5분)
2. 업무 현황 및 어려움 (15분)
3. 성장 목표 논의 (5분)
4. 매니저 피드백 제공 (5분)
```

## 작업 배분 원칙

### 공정한 배분 기준
1. 역량 매칭: 작업 요구사항 ↔ 팀원 스킬
2. 성장 기회: 도전적인 작업도 균등 배분
3. 워크로드 균형: 개인별 용량 초과 방지
4. 명확한 기대치: 결과물, 기한, 기준 명시

### 오버로드 신호
- 연속 야근 (2일 이상)
- 마감 연속 실패
- 커뮤니케이션 응답 지연
- 품질 급격한 저하

### 오버로드 대응
1. 즉시 1:1로 상황 파악
2. 현재 작업 우선순위 재정렬
3. 타 팀원에게 일부 이전 검토
4. 원인(구조적 vs 일시적) 분석

## 갈등 해소 프로세스

### 단계별 대응
1. **당사자 간 직접 해결** (1차)
   - 서로의 입장 경청
   - 공통 목표 재확인
   - 구체적 합의안 도출

2. **매니저 중재** (2차, 1차 실패 시)
   - 중립적 입장 유지
   - 사실 기반 논의
   - 합의 문서화

3. **상위 에스컬레이션** (3차, 심각한 경우)

### 갈등 예방
- 역할 및 경계 명확화
- 정기적 팀 피드백 문화
- 심리적 안전감 조성

## 팀 정렬 유지 방법

### 목표 캐스케이딩
```
Company OKR → Team KPI → 개인 목표
```
매 분기 시작 시 전체 팀과 공유

### 의사결정 투명화
- 주요 결정 사항 Notion에 기록
- 결정 이유(Why) 반드시 포함
- 영향받는 팀원에게 사전 공유

### 정보 흐름 확보
- 경영진 정보 → 팀으로 적시 전달
- 팀 이슈 → 경영진으로 적시 보고
- 정보 병목 지점 주기적 점검

## 리모트/비동기 협업

### 비동기 소통 원칙
- 모든 의사결정은 문서로
- 응답 기대 시간 명시
- 타임존 배려

### 협업 도구 사용 기준
- 동기 필요: Slack 허들 / 화상 미팅
- 비동기 가능: Slack 메시지 / Notion 댓글
- 영구 기록: Notion 문서
